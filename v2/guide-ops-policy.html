Overview
MaroPass 운영정책 가이드
일본인 한국어 학습자용 TOPIK 말하기 진단 서비스(MaroPass)의 결제·영수증·성적 등급·관리자 권한 운영 규칙을 정리한 단일 기준 문서입니다. 기능 단위 상세는 기능명세(feature-spec), 1차 오픈 라이트 정책은 docs/04-policy를 참조하세요.
핵심 원칙 3
1
일본 기준으로 운영한다
서비스는 일본에서 오픈하며 결제·영수증·세금은 모두 일본 제도(GMO-PG·適格請求書·소비세 10%)를 따른다.
2
1차 오픈은 예외를 자동화하지 않고 운영자가 수동 처리한다
자동 분기·매트릭스 같은 무거운 운영 도구는 라운드 2 이후. (→ docs/04-policy)
3
기능명세를 정본으로 한다
명세에 없는 개념(고정 급수·수료/미수료·합격/불합격)은 화면에서 쓰지 않는다.

1. 결제 · PG
QR 결제 방식
QR(코드결제)는 매장 QR스캔이 아니라 웹 리다이렉트·앱연동 방식입니다. 카드 결제와 근본적으로 다르므로 별도 처리가 필요합니다.
기기별 흐름
기기흐름
PC결제창에 QR 표시 → 휴대폰 앱으로 스캔 → 앱에서 승인
모바일앱-스위치(딥링크)로 결제앱 바로 실행 → 앱에서 승인 (QR 스캔 없음)
기술 플로우
1
결제 시작
회원이 QR 선택 → MaroPass 서버가 GMO에 거래등록·결제실행 요청 → GMO가 결제 시작 URL 반환
2
리다이렉트
브라우저를 GMO→Pay사로 이동 (PC=QR 표시 / 모바일=앱 실행)
3
승인
사용자가 결제 앱에서 승인
4
결과 회신 2채널
① 브라우저가 리턴 URL로 복귀(완료화면) ② GMO → 서버로 결과통지(Webhook)
5
확정
서버가 상태 검증 후 주문 완료 처리
결제 확정은 화면 복귀가 아니라 Webhook 기준으로 처리합니다. 사용자가 복귀하지 않아도 결제는 성립될 수 있습니다.
외부 앱 왕복이 있어 대기(pending) 상태가 존재합니다 → 결제 목록 상태 배지에 반영합니다.

2. 영수증 · 適格請求書
발행 주체
구매자용 영수증은 GMO-PG가 아니라 가맹점(MaroPass)이 직접 발행합니다.
발행 책임
GMO-PG는 구매자용 영수증을 발행하지 않습니다. GMO는 가맹점용 帳票(振込明細書·請求書·정산서)만 지원 포털에서 제공하며, 구매자 영수증은 가맹점(MaroPass)이 직접 발행합니다.

✓
영수증 모달 구현됨
어드민 payments.html · 유저 mypage-payments.html의 영수증 버튼 → 適格請求書 모달
✓
결제수단 자동 표기 변환
카드→クレジットカード, PayPay→PayPay（コード決済）, 편의점→コンビニ払い
미완/후속: 발행자 정보(회사명·주소·登録番号)는 더미 → 실제 사업자 정보 교체 필요. PDF 실제 생성은 백엔드/PG 연동 후.

3. 시험 운영
문항 스냅샷 · 출제 이력
시험에 담은 문항은 사본으로 고정되어 이후 문제은행 수정의 영향을 받지 않습니다. 버전 관리·출제 이력은 2차 예정입니다.
문항 스냅샷 1차
문항을 시험에 담는 순간 사본으로 고정(스냅샷 온 애드)됩니다. 이후 문제은행에서 원본을 수정해도 이미 담긴 시험에는 영향이 없습니다.

시험에 담긴 문항은 사본이므로 이후 문제은행의 원본·분류 변경과 무관하게 고정됩니다.

문제 분류(카테고리) 개편

문제 분류를 기존 트리(과목 > 레벨 > 유형 > 세부)에서 폴더형 사용자 분류로 개편했습니다. 관리자가 자유롭게 만든 폴더에 문항을 다중 배정하고, 레벨은 분류에서 분리해 목록 상단 필터로 둡니다. (기존 터치클래스 문제은행과 동일 방식)

항목규칙
분류 체계폴더형 사용자 정의 분류(문제 은행 좌측 트리). 예: 정기시험 1회·2회 · 모의고사 · 발음·읽기 연습 · 회화·역할극 · 화용 과제 · 미분류
레벨초급 / 중상급 — 분류 트리에서 분리해 목록 상단 필터로 이동
배정문항은 여러 분류에 다중 배정 (문제 등록·문제 은행에서 분류 선택 → 칩)
분류 관리추가(분류명 + 설명) · 삭제(소속 문항 미분류로 이동) · 드래그 순서 변경 · 검색 — 분류 관리 모달
유형별 분류축장소·상황, 화행 등 유형별 축 값은 태그로 유지(문제 은행 태그 열)
미완/후속: 위 분류명은 예시(샘플)이며, 실제 운영 분류 목록은 확정 후 반영합니다.

4. 성적 · 등급
진단 등급 체계
현재 서비스는 모의고사(진단평가)입니다. 성적 등급은 시험별로 설정하는 진단 등급(A·B·C)이며, 합격 판정·고정 급수 개념은 없습니다.
쓰는 것 / 안 쓰는 것
구분개념설명

6. 관리자 계정
부관리자 관리
관리자 계정은 유형 2종으로 나뉘며, 부관리자의 역할은 권한 설정으로 결정됩니다.
계정 생성 원칙
최고관리자 계정은 시스템에 기본으로 반드시 생성됩니다. 항상 1개 이상 존재하며 삭제할 수 없습니다. 이후 모든 관리자(부관리자) 계정은 최고관리자가 직접 추가·권한 설정합니다.
1
최고관리자 계정 기본 생성
서비스 개설 시 최고관리자 계정이 무조건 만들어집니다. 전체 권한 고정 · 삭제 불가(항상 존재).
2
최고관리자가 관리자를 세팅
이후 부관리자를 추가(임시 비밀번호 자동 발급)하고 권한 설정으로 역할을 지정합니다. 관리자 계정의 추가·권한 편집은 최고관리자만 가능.
계정 · 권한
항목규칙
계정 유형최고관리자 / 부관리자 — 최고관리자는 전체 권한 고정
역할 결정권한 설정(메뉴별 권한 체크)으로 결정 · 예: 운영형·운영+매출형·교수자형·맞춤
계정 필드이름 / 아이디 / 이메일 분리 관리 (admin-list)
추가임시 비밀번호 자동 발급 → 첫 로그인 시 변경 필수 · 72시간 내 미접속 시 만료
삭제 시 확인 모달("삭제" 입력). 삭제 대상: 계정·로그인·권한 / 유지: 작성 이력·감사 로그.
7. PG 연동 (개발 전)
PG 연동 가이드 개발 전 초안
GMO-PG(멀티페이먼트) 결제 연동을 위한 개발 기준 초안입니다. 결제 흐름·Webhook 확정·연동 타입·테스트↔운영 전환·에러 처리를 정리합니다.
⚠ 개발 전·검토 전 초안입니다. 아직 구현·연동에 착수하지 않은 단계로, 아래 내용은 GMO-PG 공개 자료(docs.gmo-pg.com/mulpay)와 운영정책을 바탕으로 한 예상 기준입니다. 가맹점 계약·연동 타입 확정 후 재검토가 필요하며, 파라미터명·정확한 상태값은 GMO 사양서 기준으로 확정합니다.
