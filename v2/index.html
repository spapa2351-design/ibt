2사이트 통합 구조 (ui2)

기존 ui/의 4개 영역(서비스·응시·서비스관리·시험관리)을 개발·운영 효율에 맞춰 학습자 사이트와 관리자 사이트 2개로 병합한 버전입니다. koen-report-v2 문서의 개발 구성 확정안을 반영합니다.

완성 · v0.1

① 학습자 사이트

서비스 사이트(홈·시험목록·상세·결제·마이페이지)를 기반으로 응시(환경점검·응시·결과)를 마이페이지 흐름 안에 통합. GNB·블루 테마·한일 i18n 공유 셸.

진입: 홈 → 시험 → 결제 · 마이페이지 → 응시 → 결과 (내부 흐름)
학습자 홈으로 →
 완성 · v0.1

② 관리자 사이트

서비스 관리(회원·결제·문의·환불) + 시험 관리(시험 목록·등록) + 문제 빌더(문제 은행·만들기)를 하나의 콘솔로 통합. 좌측 사이드바 3그룹 · 공유 셸.

진입: 관리자 로그인 → 통합 대시보드 · 10개 화면
관리자 로그인 →
참고 · 기존 ui/ 폴더는 그대로 보존되며, 이 ui2/가 병합 결과입니다. 두 사이트 모두 공유 셸 아키텍처를 씁니다 — 학습자는 ui2/user/(assets/user.css + user-shell.js, GNB·푸터·한일 i18n 주입), 관리자는 ui2/admin/(admin.css + admin-shell.js, 사이드바 3그룹). 응시 흐름 변경: 기존 "새 창" 방식 대신 마이페이지 → 환경점검 → 응시 → 결과가 같은 사이트 내부에서 이어집니다.
