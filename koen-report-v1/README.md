# KoPASS IBT 기획문서 v1 — KOEN 보고용

> 코엔(코엔, 일반사단법인) 측 공유용 기획 패키지. Newin 내부 SSOT(`spec.html`, `feature-spec.html`)와는 별개의 *외부 보고용* 문서입니다.

## 시스템 구조 (3개 영역)

1. **서비스 사이트 (KoPASS)** — 가입·로그인, 시험 목록, 결제, 환불, 1:1 문의, 마이페이지, **인증서·결과지 PDF 다운로드** 등 회원·결제 영역
2. **터치클래스 사용자 web (새창)** — 결제된 시험 목록, 응시 진행, 응시 결과 확인. 서비스 사이트의 "응시하기" 버튼 클릭 시 **새 창**으로 진입
3. **터치클래스 관리자 web** — 기존 솔루션 메뉴 + 신규 **회원 목록** 메뉴(서비스 사이트의 회원가입·결제 데이터 모음)

> **1차 오픈 디바이스**: 데스크탑·모바일 **웹**. iOS/Android 네이티브 앱은 라운드 2 이후 도입.
> **응시 진입 정책**: 응시는 사용자 web이 *새 창*으로 열림. 결과 확인은 사용자 web, PDF 다운로드는 서비스 사이트 마이페이지로 분리.

## 패키지 구성

| 파일 | 용도 |
|---|---|
| `index.html` | 통합 보고서 — 서비스 개요·시스템 구조·응시자 플로우·운영 정책·역할 분담·진행 단계·향후 계획 (인쇄/PDF 가능) |
| `flow-overview.html` | 플로우 박스 다이어그램 — 응시자 5단계 + KOEN 운영자 플로우를 한눈에 (박스+화살표, 종이 시안 톤) |
| `wireframes.html` | 화면별 와이어프레임 — 사이드바 네비로 3시스템 13개 화면 전환(서비스 사이트 7 + 사용자 web 5 + 관리자 web 1), 한/일·데스크탑/모바일 토글 지원 (종이 시안 톤, 실제 UI 아님) |

## 읽는 순서 (권장)

1. **`index.html`** — 통합 보고서로 사업 윤곽 + 시스템 구조 파악
2. **`flow-overview.html`** — 응시자가 어떤 단계를 거치는지 한눈에
3. **`wireframes.html`** — 화면별로 어떤 요소가 들어가는지 디테일

## 사용 안내

- **이 문서는 1차 오픈 기획 v1**입니다. UI 디자인·구현 결과물은 별도이며 변경될 수 있습니다.
- 와이어프레임은 *흐름 확인용 박스 다이어그램*입니다. 실제 화면 디자인이 아니므로 색상·레이아웃·문구는 추후 변경됩니다.
- 단계·범위·정책 변경은 본 패키지의 차기 버전(v2 등)으로 별도 발행됩니다.
- 내부 GAP 번호·라벨·코드 레퍼런스는 본 패키지에서 제거되었으며, 필요 시 Newin 측에 별도 요청 가능합니다.
