01 / Product서비스 개요

학습자가 받게 되는 가치와 시험의 구조.

학습자가 받는 것

  • 발음·유창성·어휘·문법·내용 구성 5개 영역의 점수표
  • 5축 시각화 5각 레이더 차트(리포트 부가)
  • 문항별 AI 피드백 — 답안과 STT 비교, 추천 연습 안내
  • 1급~6급 진단 등급과 종합 코멘트
  • 결과지 PDF와 인증서 PDF, 위변조 검증 URL 포함

시험 구조

시험 시간약 25분 (23문항)
5개 영역발음 · 유창성 · 어휘 · 문법 · 내용 구성
채점AI 1차 자동 채점. 시험 종료 직후 처리. 운영자 보정 가능
결과 공개운영자가 관리자 콘솔에서 공개 처리 (정책 5)

TOPIK과의 관계

KoPASS는 TOPIK의 대체가 아닙니다. TOPIK 말하기 시험을 보완하는 진단·모의고사 포지션이며, 학습자가 자신의 말하기 실력을 객관적으로 점검할 수 있도록 돕습니다. 메인 페이지와 약관에 본 포지셔닝을 명시합니다.

02 / Flow응시자 동선

학습자가 처음 만나서 인증서를 받기까지 다섯 단계.

STEP 1
진입
가입 · 로그인

이메일 인증 가입 또는 LINE·Google·Apple 소셜 가입. 일본 시장 표준 3종만 지원(카카오·네이버 미지원). 약관 동의와 최소 개인정보 수집.

STEP 2
결제
시험 선택 · 결제

시험 목록에서 시험 상세로 진입해 가격·구성·샘플 문항을 확인한 뒤 결제. 신용카드 또는 QR 결제. 결제 완료 메일과 영수증 PDF가 자동 발행됩니다. 적격청구서 인보이스는 PG API 연동 시점에 함께 도입 예정.

STEP 3
대기
응시 대기 (14일 윈도우)

마이페이지의 나의 시험에서 D-day를 확인합니다. 결제일로부터 14일 이내 24시간 언제든 응시 가능. D-1과 D-day에 자동 리마인더 메일을 발송하며, 14일 경과 시 자동 만료됩니다.

STEP 4
응시
응시 — 본인 확인 → 환경 점검 → 시험

시험 시작 버튼을 누르면 ① 본인 확인(이름·이메일·시험명 표시), ② 환경 점검 게이트(마이크·네트워크·브라우저), ③ 응시 클라이언트 진입 순으로 진행되며 종료 직후 AI가 자동 채점합니다.

STEP 5
결과
결과 · 인증서

운영자가 결과를 공개하면 학습자는 마이페이지에서 점수표, 5각 레이더, 문항별 AI 피드백을 확인할 수 있습니다. 결과지 PDF와 인증서 PDF(위변조 검증 URL 포함)를 무기한 재다운로드 가능.

동선 박스 다이어그램은 flow-overview.html, 화면별 와이어프레임은 wireframes.html에 별도로 정리되어 있습니다.

03 / Policy운영 정책

학습자가 막혔을 때 일어나는 일과 운영팀이 다뤄야 할 영역.

응시 윈도우와 만료

  • 응시 가능 기간 — 결제 후 14일 이내
  • 만료 알림 — D-1과 D-day에 자동 메일 발송
  • 만료 후 — 자동 만료 처리. 환불은 제공되지 않으며 재응시도 불가
  • 예외 — 시스템·네트워크 장애로 응시 진행이 불가했음이 확인된 경우 운영자 수동 검토 후 재응시 티켓 발급

환불 · 취소

  • 응시 전 미사용 환불 신청은 마이페이지 1:1 문의로 접수. 운영팀 검토 후 영업일 3~5일 이내 환원
  • 응시 후 변심·결과 불만족은 환불 불가
  • 시스템 장애로 응시 진행 불가 시 자동 환불 또는 재응시 티켓 발행 — 운영자 판단

응시 본인 확인

응시 시작 직전 본인 확인 화면에서 이름·이메일·시험명을 표시합니다. 학습자가 본인임을 체크박스로 확인한 뒤 환경 점검 게이트로 진입합니다. 카메라·SMS 검증은 추후 도입 예정.

환경 점검 게이트

본인 확인을 통과한 직후 마이크 권한, 녹음 재생, 네트워크, 브라우저 4개 항목을 자동 점검합니다. 통과 시에만 응시 클라이언트에 진입할 수 있습니다. 결제 시점과 분리되어 있어 결제는 환경과 무관하게 진행됩니다.

응시 중 사고 대응

  • 네트워크 끊김 — 자동 재연결, 시간 보존
  • 마이크 권한 회수 — 알림과 재허가 가이드
  • 타이머 — 카운트다운 후 답안 자동 제출
  • 이탈 차단 — 응시 중 페이지 닫기·뒤로가기 시 경고
  • 지속적 오류 — 운영팀에 자동 로그 전송, 재응시 티켓 검토

시스템 알림 메일

시점내용
결제 완료 직후결제 확인 + 영수증 PDF 안내
응시 윈도우 D-1 / D-day응시 기간 임박 리마인더
응시 윈도우 만료자동 만료 안내, 향후 응시 안내
결과 공개 시운영자 공개 처리에 따라 자동 발송
환불 처리 완료환원 완료 안내

결과·인증서 보관

결과와 인증서는 무기한 보관되며 재발급 횟수에 제한이 없습니다. 학습자는 마이페이지에서 언제든 다시 다운로드할 수 있습니다.

1:1 문의 동선

마이페이지의 1:1 문의에서 유형(결제·시험·오류·환불 등)을 선택하고 제목, 내용, 첨부파일을 작성합니다. 운영팀이 답변하면 학습자에게 메일이 자동 발송되며 상태는 접수 / 답변완료 두 단계로 관리됩니다.

05 / Stages진행 단계

초안 기준 단계 흐름. 구체적인 일정·날짜는 본 패키지 합의 이후 별도 합의로 확정합니다.

STAGE 1
기획 v1 확정
서비스·동선·정책·역할 분담 합의. 본 문서가 기준점입니다.
STAGE 2
디자인 시안과 개발 착수
화면별 디자인 발주와 검수. 시스템 개발이 본격적으로 시작됩니다.
STAGE 3
베타 오픈
제한된 응시자 풀에 시범 운영. 응시 흐름·결제·결과 동선을 검증합니다.
STAGE 4
베타 피드백 반영 · 운영 리허설
KOEN 운영팀이 응시자 문의·환불·결과 공개 동선을 시뮬레이션합니다.
STAGE 5
정식 서비스 개시
일반 응시자 대상 오픈.

06 / Future향후 계획

1차 오픈 후 운영 데이터를 보고 결정할 영역. 본 패키지에서는 예정으로만 표시합니다.

결제·결제 수단 확장
적격청구서 인보이스 PDF 도입(일본 소비세법 대응), 쿠폰·할인 코드, 이용권 패키지 결제.
운영 통계 확장
응시자 분석(평균 발화 길이·이탈률), 문항별 응답 시간 분포, AI 채점 vs 운영자 보정 차이 분석.
문제 만들기 깊이
LLM 자동 생성 보조(지문·TTS 스크립트·모범 답안·채점 기준표), 문제 품질 평가 자동화, 다국어 문제 동시 작성.
보안 · 본인 확인 강화
응시자 2FA, 카메라·SMS 본인 확인, IP 화이트리스트, 권한 매트릭스 UI.
알림 채널 확장
앱 푸시 알림 인프라, 알림 종류별 수신 설정 분리.
시험 운영 확장
쓰기·듣기·읽기 영역 추가, 다국어 시험지 동시 운영, 시험 통과 조건 세분화.

07 / Scope본 문서가 다루지 않는 것

  • 화면 디자인 — 와이어프레임은 흐름 확인용 박스 다이어그램이며, 실제 UI 디자인은 별도 진행됩니다
  • 기술 사양 — 데이터 모델·API·인프라 세부 사항은 Newin 내부 문서에서 다룹니다
  • 가격 정책 확정 — 시험별 단가·할인가 정책은 KOEN과 별도 합의 후 운영 직전 확정합니다
  • 관리자 권한 상세 — KOEN 운영자별 권한 매트릭스는 별도 합의 문서로 발행됩니다
  • 법적 약관 문구 — 이용약관·개인정보처리방침·특정상거래법 표시 문구는 법무 검토 후 별도 확정합니다
