# 기능 명세 (Phase 1, Light)

> 한국 LMS 표준 포털 기능 스펙을 *일본 시장 + 1차 오픈 light*에 맞게 정정한 라인 아이템 매트릭스.

## 한 줄

**5대분류 × 11중분류는 그대로 두되, 한국 PG·쿠폰·이용권·장바구니 같은 무거운 영역은 모두 DEFERRED로 잘라낸다.**

## 이 한 줄을 지지하는 이유 5개

1. 한국 LMS 표준 패턴(카카오/네이버 OAuth, 이용권 패키지, 쿠폰·포인트, 가상계좌)은 일본 시장과 PG·소비자 행동이 달라 그대로 박으면 1차 오픈 일정·정합성이 흔들린다.
2. **단건 결제 + 일본 PG 1개**라는 *light* 가정을 지키면 결제·환불·트랜잭션 정합성 표면적이 1/3 수준으로 줄어든다.
3. 1차 오픈에서 자르는 모든 항목은 라운드 2에서 *운영 데이터 + 응시자 패턴*을 보고 결정해야 손해가 없다.
4. *대-중분류*는 응시자 멘탈 모델(메인·회원·구매·마이페이지·고객센터)이라 변경 가능성이 낮으므로 골격은 유지한다.
5. kopass 특수(환경 점검 게이트·인증서 위변조 검증·5각 레이더 차트·인보이스 PDF)는 한국 LMS 스펙에 없는 항목이라 별도 표시한다.

## kopass 특수 (한국 LMS 스펙에 없던 항목)

1. **환경 사전 점검 게이트** — 응시 시작 직전 (마이페이지 > 나의 시험 > 시험 시작 → 게이트) (정책 1, GAP-08). 결제와 분리
2. **응시 본인 확인** — 응시 시작 직전 이름·이메일·시험명 확인 (카메라·SMS는 라운드 2)
3. **응시 클라이언트 진행 정책** — 네트워크 끊김 자동 재연결·마이크 권한 회수 가드·타이머·답안 자동 제출·이탈 차단 (정책 4 확장)
4. **TOPIK과의 관계 안내** — 보완·진단 포지셔닝 (메인·약관)
5. **결과 — 점수표 (선) + 5각 레이더 (리포트 부가)** — 발음·유창성·어휘·문법·내용 구성
6. **AI 피드백 (종합 + 문항별)** — STT 비교·추천 연습
7. **결과지 PDF + 인증서 PDF + 위변조 검증** — 다운로드 + 외부 URL 검증
8. **응시 진입 — 데스크탑 / 앱 분기 안내** — 단일 웹 앱 + 네이티브 쉘 구조 (GAP-04, GAP-14)
9. **응시 윈도우 14일 티켓 + 만료 처리** — 정책 8. 만료 D-1·D-day 메일, 만료 후 환불 정책 매핑
10. **시스템 알림 메일 5종** — 결제 / D-day / 만료 임박 / 결과 공개 / 환불 처리

> **인보이스 PDF (일본 적격청구서)**: 일본 PG API 연동 시점에 같이. 라운드 2.
