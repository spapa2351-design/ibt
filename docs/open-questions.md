상태 표기: `OPEN` / `BLOCKED` (외부 답 대기) / `DEFERRED` (1차 오픈 범위 외)

- 🟡 **Minor** — 영향은 있지만 가정 또는 우회로 진행 가능

## 갭 상세

### GAP-01. 등급 체계 — 🔴 Critical

- **상태**: OPEN
- **충돌**:
  - 마스터 문서: *"종합 결과 — 진단 등급 (A~D 4단계)"*
  - 시험 결과 디자인: *"등급 1급 / 6급"*
  - AI 피드백 디자인: *"B / Good"* (문항별, A~F 추정)
  - 응시자 관리 백오피스: *"1급"* 표시 + *"등급별"* 필터
- **가설**: 두 트랙 분리 — **종합 진단 등급**(1~6급, TOPIK 매핑)과 **문항별 학습용 피드백 등급**(A~F)
- **막히는 것**: 성적표 디자인, AI 채점 출력 스펙, 시험 통과 조건 정의

### GAP-02. 말하기 영역 정의 — 🟢 RESOLVED (초급)

- **상태**: RESOLVED (초급), OPEN (중상급)
- **해소**: `re/251020_speaking_test_beginner_ko_re3.docx`에 초급 5영역(짧은 문장 읽기·질문 대답·생활 속 대화·그림 설명·화용 과제) 정의 확인. 영역 6은 ver.3에서 삭제됨 (0·1·2·3·4·5·7 번호 체계).
- **반영**: `01-glossary.md` 문제 유형 섹션 + `06-admin-feature-spec.md` 문제 만들기 섹션 + `ui-admin-question-create.html` 5영역 탭 UI.
- **남은 것**: 중상급 6영역 구성. 박천홍·하정일 교수 미팅에서 확정 필요. 그 전까지 admin UI는 초급 5영역만 노출.

### GAP-03. 초급 23문항 vs 디자인 6문항 — 🟠 Important

- **상태**: OPEN
- **충돌**: 마스터 문서 — 초급 23문항 / 중상급 14문항 vs 디자인의 6문항(200점)
- **추정**: 디자인은 베타 테스트용 축약본 또는 데모일 가능성
- **막히는 것**: 시험 시간(초급 25분 / 중상급 35분), UI 페이지네이션, 자동 저장 빈도

### GAP-04. 네이티브 쉘 기술 스택 — 🟠 Important

- **상태**: OPEN
- **유력 후보**: Capacitor (Ionic) — Web 코드 그대로, 네이티브 플러그인 추가, iOS/Android 동시
- **대안**: Cordova(구식), React Native WebView(어색), PWA only(앱스토어 제약)
- **막히는 것**: 빌드 파이프라인·배포 절차·플러그인 선택지

### GAP-05. 마이크·녹음 — Web API vs 네이티브 fallback 기준 — 🟠 Important

- **상태**: OPEN (검증 필요)
- **검증 포인트**:
  - iOS WebView의 `MediaRecorder`·`getUserMedia` 안정성
  - 백그라운드(앱 스위치·잠금) 녹음 지속 여부
  - 인터럽션(전화·알람·블루투스 끊김) 처리
  - 권한 요청 UX (네이티브 다이얼로그 vs WebView 안 다이얼로그)
- **막히는 것**: 베타(2026.7) 전 검증 일정·환경 확보. fallback 발동 기준 정량화.

### GAP-06. 2차 채점 운영 시점 — 🟡 Minor

- **상태**: OPEN
- **자료**: 마스터 문서 — *"1단계(AI): 시험 종료 즉시 / 2단계(Human): 평가자 검수"*
- **1차 오픈 가정**: AI 1차 채점만. 2차 평가자 검수는 라운드 2 이후 (Premium 도입 시).
- **막히는 것**: 평가자 UI 디자인 발주 시점, 백오피스 권한 모델

### GAP-07. 평가 차원 5개 vs 6개 — 🟡 Minor

- **상태**: OPEN
- **5개 (영역별 차트)**: 발음·유창성·어휘·문법·내용 구성
- **6개 (AI 채점 알고리즘)**: 발음 정확도·발화 속도·유창성·어휘·문법·내용 구성
- **추정**: "발화 속도"는 "유창성"의 하위 항목일 가능성. 단순 표현 차이.

### GAP-08. 환경 점검 사후 책임 한계 — 🟠 Important

- **상태**: OPEN (정책)
- **마스터 문서**: 사전 테스트 통과 없으면 결제 버튼 비활성화. 통과 후 개인 기기 문제는 사용자 책임.
- **막히는 것**: 사전 테스트와 본 시험 사이 시간 갭 처리 — 응시 시작 직전 재테스트 의무화 여부.

### GAP-09. 응시자 메모장 데이터 처리 — 🟡 Minor

- **상태**: OPEN
- **마스터 문서**: 준비 시간에 메모장 활성화 (키보드 입력)
- **1차 오픈 가정**: (a) 응시자만 보는 임시 데이터 — 일반 IBT 관행. 평가자·AI 채점에는 미반영.

### GAP-10. 일본어 vs 한국어 UI (i18n) — 🟠 Important

- **상태**: OPEN
- **타깃**: 일본인 학습자 → 응시 UI 일본어
- **시험 콘텐츠**: 한국어 (문제·답안)
- **막히는 것**:
  - 시험 진행 메시지("녹음 중", "준비 00:20")는 일본어, 문제는 한국어 — 폰트·줄바꿈 분리
  - 관리자 백오피스 언어 (한국어 vs 일본어) 미정

### GAP-11. 데스크탑 뷰포트 응시 UI 디자인 부재 — 🔴 Critical

- **상태**: OPEN (디자인 발주)
- **컨텍스트**: 데스크탑이 1순위 디바이스. 클라이언트는 단일 반응형 웹 앱이므로 별도 앱이 아니라 **반응형의 데스크탑 뷰포트 디자인**이 필요
- **자료 현황**: 본 디자인 자료의 응시 화면은 모두 모바일 1단 레이아웃
- **마스터 문서**: 좌측(문제 영역) / 우측(응시 영역) 2단 분할 명시
- **막히는 것**: 반응형 응시 화면 디자인 — 모바일 1단(있음) + 데스크탑 2단(부재) + 중간 브레이크포인트

### GAP-12. 결제 PG사 미확정 — 🔴 Critical

- **상태**: BLOCKED (외부 협상)
- **자료**: "일본 PG사 1개 연동 예정"
- **막히는 것**: 결제 수단(신용카드만? QR 포함?), 환불 트랜잭션 흐름, 영수증 포맷, SDK·API 사양
