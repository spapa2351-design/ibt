# Phase 로드맵

## 한 줄

**1차 오픈(Phase 1)은 *말하기 진단 + 가입·접수·결제·응시·결과·인증서* 한 동선으로 끝낸다. 그 외는 라운드 2 이후.**

## 이 한 줄을 지지하는 이유 5개

1. 데드라인이 **2026년 9월 서비스 개시**로 박혀 있어 범위 확대는 일정 위험으로 직결된다.
2. 일본 시장 응시자는 안정성·예측 가능성을 중시하므로 *기능 적게·정확히*가 *기능 많이·불안정*보다 가치가 크다.
3. 관리자 측은 기존 솔루션을 활용하므로 1차 오픈은 *대메뉴 식별*만으로 충분하고, 향후 핵심인 *문제 만들기*는 라운드 2 이후로 미룬다.
4. 평가자(2차 채점)·딥링크·푸시 알림·장바구니 같은 무거운 기능은 1차 오픈 후 운영 데이터를 보고 결정하는 게 합리적이다.
5. 라운드를 작게 끊으면 갭 결정이 빠르게 본문에 반영되어 문서가 늘 최신 상태로 유지된다.

## Phase 1 — 1차 오픈 (2026.9 목표)

### 응시자 (서비스 웹사이트 + 앱)

| 영역 | 1차 오픈 포함 |
|---|---|
| 회원관리 | 가입 / 로그인 / 비밀번호 재설정 / 계정 찾기 |
| 상품·결제 | 시험 목록·상세, 일본 PG 1개 연동 결제 (신용카드 + QR 1종) |
| 마이페이지 | 구매 시험·응시·결제 내역·환불 신청·인증서 다운·정보 수정 |
| 응시 클라이언트 | 환경 점검·오리엔테이션·응시(자동 녹음·자동 저장)·결과 |
| 결과·피드백 | AI 1차 채점 결과·5각 레이더·문항별 AI 피드백·인증서 PDF |
| 고객센터 | 공지·FAQ·1:1 문의 |
| 알림 | 이메일만 (채점 완료·결제 완료) |
| 디바이스 | 데스크탑 브라우저 + iOS/Android 앱(WebView 래퍼) + 모바일 웹(폴백) |

### 관리자 (백오피스)

| 영역 | 1차 오픈 포함 |
|---|---|
| 시험 관리 | 시험 등록·공개·응시 조건 설정 |
| 평가/채점 | 응시자 결과 조회·답안 초기화·결과 공개 (AI 1차 채점 자동) |
| 회원/권한 | 회원 조회·기본 등급별 권한 |
| 통계 | 응시 통계 기본 |
| 시스템 설정 | 인증서 템플릿·약관·2FA·활동 로그 |

> 관리자는 기존 솔루션 활용 + 대메뉴 식별만. **문제 만들기**는 향후 핵심으로 별도 라운드.

### UI 공통 적용

- 한국어/日本語 i18n 토글 (localStorage 동기화 + `data-i18n` / `data-i18n-html` 패턴)
- 햄버거 메뉴 (≤768px) + 풀너비 드로어 + 좌우 20px 패딩
- 반응형 베이스 리셋 (`*{box-sizing}` + `html,body{overflow-x:hidden;max-width:100vw}` + `img,svg{max-width:100%}`)
- 페이지 간 링크 전체 연결 (GNB·CTA·sidebar·breadcrumb)
- 일관 톤 (마케팅·홈은 그라디언트 / 응시·결제는 차분)
- 응시 화면 1080px shell 가운데 정렬 + 상하 70/30 분할
- 메인 홈 FAQ 5개 아코디언 + 한·일 답변 더미 + `<strong>` 강조
- 로그인 진입은 `ui-auth.html#login` 해시로 자동 탭 활성화
