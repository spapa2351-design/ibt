# 관리자 기능 명세 (Phase 1, Light)

> 관리자는 기존 솔루션을 활용하므로 1차 오픈은 *대메뉴 식별 + 핵심 워크플로우 골격*만. 향후 핵심인 **문제 만들기**는 별도 섹션에 강조.

## 한 줄

**관리자 1차 오픈은 시험 등록·응시자 결과 공개·기본 통계·시스템 설정의 골격만 잡는다 — 문제 만들기와 평가자 UI는 라운드 2 이후 깊이.**

## 이 한 줄을 지지하는 이유 5개

1. 사용자가 *관리자는 기존 솔루션 활용*이라고 못 박았으므로, 1차 오픈에서 관리자 측에 시간을 쏟으면 응시자 측 일정이 흔들린다.
2. 관리자 워크플로우는 *시험 등록 → 응시자 결과 조회 → 결과 공개*의 한 동선만 막히지 않으면 9월 런칭이 가능하다.
3. **문제 만들기**는 LLM 자동 생성·문제 은행·Drag & Drop 시험지 구성 등 깊이가 큰 영역으로, 사용자가 *향후 핵심*이라 한 만큼 별도 라운드에서 깊게 다뤄야 한다.
4. 평가자 UI(3단 분할 뷰·STT 싱크·매크로·2인 교차 채점)는 Premium 플랜 도입 시점에 종속된 영역이므로 1차 오픈 비포함.
5. 통계·분석 영역은 운영 데이터가 충분히 쌓인 후에 디자인하는 게 합리적이라 1차 오픈은 *기본 카운트*만 보여준다.
