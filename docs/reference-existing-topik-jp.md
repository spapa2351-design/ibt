# 참고 — 현행 TOPIK 일본 사이트 분석

> 본 문서는 **참고 자료**다. KoPASS의 결정 사항이 아니다. 현 TOPIK 일본 응시자가 익숙한 동선을 이해하기 위해 정리한다.

## 한 줄

**현 TOPIK 일본 사이트는 "마케팅 사이트 + 회원/접수 시스템"이 도메인 분리되어 있다 — 응시자는 안내사이트에서 정보를 보고 별도 도메인에 로그인해서 접수한다.**

## 도메인 구성

| 도메인 | 역할 | 기술 단서 |
|---|---|---|
| `kref.or.jp/topik/` | 마케팅·안내·공지·FAQ | WordPress (`wp-content`, 테마: `korea-edu`) |
| `topikjp.kref.or.jp/main/signin/` | 회원가입·로그인·접수·마이페이지·환불 | (HTML 비공개, JS SPA로 추정) |
| `topik.go.kr` | 성적 확인 | 한국 본사 시스템 |

> 일본 측은 **접수까지** 책임지고, 성적은 한국 본사로 리다이렉트.

## 안내 사이트 메뉴 구조 (일본어)

```
TOPIKについて
 ├─ 試験概要
 ├─ 歴史
 ├─ 主管機関紹介
 ├─ 資格の活用
 └─ CEFR比較表

受験について
 ├─ 実施日程・会場
 ├─ 受験申請
 ├─ 試験に関する注意事項
 └─ Q&A

韓国留学・奨学金
 ├─ 奨学金概要
 └─ GKS사업

関連サービス
 ├─ 資料ダウンロード
 ├─ 申請内容変更
 ├─ 受験料返金
 └─ 書類・証明書発行
```

상단 바 직링크: `受験申請` · `受験料返金` · `申請確認・変更` · `成績確認`

> 직링크 4개가 모두 회원/접수 도메인(`topikjp.kref.or.jp`)으로 빠진다. 즉 안내 사이트는 거의 모든 트랜잭션을 별도 도메인에 위임.

## KoPASS에 시사하는 것

1. **응시자 동선 가설** — 안내 사이트와 트랜잭션 사이트 분리 패턴이 일본 응시자에게 익숙하다. KoPASS도 단일 사이트 vs 분리 패턴 결정 필요.
2. **앱이라는 신호** — 현 TOPIK엔 앱이 없다. KoPASS의 **"가입 → 마이페이지 → 앱 오픈"** 동선은 차별화 포인트. 마이페이지에서 앱 오픈을 유도하는 진입점 설계가 핵심.
3. **앱의 역할 가설** — 앱은 응시 전용일 가능성이 높다. 회원가입·결제·마이페이지는 웹 단독으로 두면 PG·OS 정책 변경 영향이 줄고, 앱은 시험 응시 환경(마이크 권한·녹음 안정성)에 집중할 수 있다.
4. **i18n** — 안내·접수 모두 일본어. KoPASS도 동일 정책. 시험 콘텐츠(한국어)와 분리 설계는 변하지 않음.
5. **성적/인증서** — 현 TOPIK은 한국 본사로 빠지지만, KoPASS는 일본 단독으로 성적·인증서까지 책임진다. 즉 KoPASS의 마이페이지가 현 TOPIK 마이페이지보다 **더 무거워진다** (응시 이력 + 성적 + 인증서 다운로드 + 재응시 등).
