# 정보 구조 (IA — Information Architecture)

> 서비스 웹사이트(메인 사이트)의 *페이지 계층 + 라벨 + 네비게이션 + 분류 체계*. 디자인 발주 wireframe의 출발점.

## 한 줄

**모든 페이지가 *시험 → 응시 → 결과 → 인증서* 한 흐름으로 흐르도록, 다른 콘텐츠는 보조 위치에 배치한다.**

## 이 한 줄을 지지하는 이유 5개

1. 1차 오픈의 한 동선(가입 → 접수 → 결제 → 응시 → 결과 → 인증서)이 매출과 사용자 가치 모두의 핵심이므로 모든 페이지가 이 흐름의 진입·복귀를 지원해야 한다.
2. 일본인 응시자는 예측 가능한 네비게이션을 선호하므로 *글로벌 메뉴 위치·하단 탭 위치·셋오프 위치*가 일관되어야 한다.
3. 한국 LMS 표준 GNB(시험·이용권·고객센터·마이페이지) 4메뉴를 *이용권 제외*해 3메뉴로 줄여야 1차 오픈 light 정합성이 유지된다.
4. 모바일·앱은 하단 탭이 엄지 접근성에 유리하고, 데스크탑은 상단 GNB가 표준이므로 디바이스별로 글로벌 네비게이션을 분리한다.
5. 라벨은 일본어 1차로 박되 한국어 매핑(시험 콘텐츠용)을 별도로 두면, i18n 분기 코드와 디자인 발주 사양이 흔들리지 않는다.

---

## 1. IA 원칙 5개

| # | 원칙 | 의미 |
|---|---|---|
| 1 | **단일 동선 우선** | 시험·응시·결과·인증서가 메인 경로. 다른 모든 페이지는 보조 |
| 2 | **2클릭 룰** | 메인 행동(응시 시작·결과 확인)은 어디서든 2클릭 이내 도달 |
| 3 | **셋오프 레이아웃** | 회원/계정 영역은 우상단 셋오프. 핵심 GNB는 콘텐츠 중심 |
| 4 | **모바일 하단 탭 / 데스크탑 상단 GNB** | 디바이스별 표준 패턴 분리. 동일 라벨·동일 진입 동선 유지 |
| 5 | **라벨 일본어 1차** | UI 라벨 = 일본어, 시험 콘텐츠 영역(`:lang(ko)`)만 한국어 |

## 2. 글로벌 네비게이션

### 2.1 데스크탑 GNB (상단)

```
┌───────────────────────────────────────────────────────────────┐
│ [로고]   試験一覧   マイページ   サポート     [로그인/プロフィール]│
└───────────────────────────────────────────────────────────────┘
       └─ 좌측 정렬 ──┘  └─ 중앙 핵심 GNB 3개 ─┘  └─ 우상단 셋오프 ─┘
```

| 위치 | 일본어 (가설) | 진입 |
|---|---|---|
| 좌측 | 로고 | `/` (홈) |
| 중앙 | **試験一覧** | `/tests` |
| 중앙 | **マイページ** | `/me` |
| 중앙 | **サポート** | `/support` (FAQ + 1:1) |
| 우상단 | (비로그인) ログイン / 会員登録 | `/signin`, `/signup` |
| 우상단 | (로그인) 프로필 아바타 + 드롭다운 | 마이페이지·로그아웃 |

### 2.2 모바일·앱 하단 탭 (4개)

```
┌────────────────────────────────────────────┐
│              (콘텐츠 영역)                  │
├────────────┬────────────┬───────┬──────────┤
│  ホーム    │  試験     │ マイ   │ サポート│
│  Lucide    │  Lucide    │ Lucide│ Lucide   │
│  Home      │  ListChecks│ User  │ HelpCircle│
└────────────┴────────────┴───────┴──────────┘
```

| 탭 | 일본어 (가설) | 진입 | 활성 색 |
|---|---|---|---|
| 1 | **ホーム** | `/` | `blue-500` 아이콘 + `grey-900` 라벨 |
| 2 | **試験** | `/tests` | 동일 |
| 3 | **マイ** | `/me` | 동일 |
| 4 | **サポート** | `/support` | 동일 |

> **회원가입·로그인**은 모바일에서도 우상단 셋오프(또는 헤더 우측). 하단 탭에 넣지 않음.

### 2.3 푸터 (모바일·데스크탑 공통)

| 영역 | 항목 |
|---|---|
| 정책 | 이용약관 / 개인정보처리방침 / **特定商取引法** (일본 필수) |
| 회사 정보 | 사업자 정보 |
| SNS | X / Instagram / YouTube (현 TOPIK JP 사이트 패턴) |
| 플로팅 (모든 페이지) | [문의·FAQ] 버튼 — 우하단 |

## 4. 페이지별 콘텐츠 블록 (디자인 발주 wireframe 출발점)

### 4.1 홈 `/`

```
┌─ Hero 메인 배너 (롤링) ─────────────────┐
│  3~5장, autoplay 5s, dot indicator        │
└──────────────────────────────────────────┘
┌─ TOPIK과의 관계 안내 ───────────────────┐
│  보완·진단 포지셔닝 1 paragraph + CTA    │
└──────────────────────────────────────────┘
┌─ 추천 시험 (3~6장 카드) ────────────────┐
│  베스트·신규·인기 라벨                   │
└──────────────────────────────────────────┘
┌─ 서비스 소개 (AI 평가 방식) ────────────┐
│  3 columns: AI 채점 / 환경 점검 / 인증서 │
└──────────────────────────────────────────┘
┌─ FAQ 발췌 (5~8개) ──────────────────────┐
│  Accordion 또는 카드                     │
└──────────────────────────────────────────┘
```

### 4.2 시험 상세 `/tests/:slug`

```
┌─ 시험 헤로 ─────────────────────────────┐
│  표지 이미지 + 시험명 + 가격 + CTA       │
└──────────────────────────────────────────┘
┌─ 좌: 콘텐츠 / 우: 구매 사이드바 (sticky) ┐
│ ┌─────────────┐  ┌──────────────────┐  │
│ │시험 소개     │  │ 가격             │  │
│ │문항 구성     │  │ 응시 기간 (14일) │  │
│ │기대 효과     │  │ 응시 시간        │  │
│ │샘플 문항     │  │ 응시 환경        │  │
│ │AI 평가 방식  │  │ [購入する]       │  │
│ └─────────────┘  └──────────────────┘  │
└──────────────────────────────────────────┘
모바일: sticky CTA 하단 고정
```

### 4.3 마이페이지 대시보드 `/me`

```
┌─ 환영 인사 + 진행 중 시험 알림 ─────────┐
│  최근 결과 발표·미응시 시험 D-day 표시   │
└──────────────────────────────────────────┘
┌─ 빠른 진입 (4 카드) ────────────────────┐
│  受験する  / 結果   / 証明書 / 決済履歴  │
└──────────────────────────────────────────┘
┌─ 응시 가능한 시험 (리스트) ─────────────┐
│  14일 윈도우 D-day 표시                  │
│  [応시하기] CTA — 데스크탑/앱 분기 안내  │
└──────────────────────────────────────────┘
┌─ 최근 결과 (1~3장) ─────────────────────┐
│  점수·등급·결과 보기 CTA                 │
└──────────────────────────────────────────┘
```

### 4.4 결과 화면 `/me/results/:id`

```
┌─ 헤더 (시험명·응시일·종합 등급) ─────────┐
└──────────────────────────────────────────┘
┌─ 종합 점수 (Display 30px+) ─────────────┐
│  30점 / 200점 만점 + 1~6급 등급 (GAP-01) │
└──────────────────────────────────────────┘
┌─ 5각 레이더 차트 ───────────────────────┐
│  발음·유창성·어휘·문법·내용 구성         │
└──────────────────────────────────────────┘
┌─ AI 종합 피드백 (코멘트 1~2단락) ───────┐
└──────────────────────────────────────────┘
┌─ 문항별 점수 + AI 피드백 진입 (리스트) ─┐
│  각 문항 행: 점수 + [Ai 피드백] 모달     │
└──────────────────────────────────────────┘
┌─ CTA ──────────────────────────────────┐
│  [証明書ダウンロード] [問題を見直す]      │
│  (재응시는 시스템 장애 케이스만 — 정책 6) │
└──────────────────────────────────────────┘
```

## 5. 분류 체계 (Taxonomy)

### 5.1 시험 카테고리

| 분류 축 | 값 (Phase 1) |
|---|---|
| **레벨** | 초급 (Beginner, TOPIK I 수준) / 중상급 (Intermediate-Advanced, TOPIK II 수준) |
| **영역** | 말하기 (Phase 1 단독) |
| (라운드 2) 영역 확장 | 쓰기 / 듣기 / 읽기 — Phase 1 미적용 |

### 5.2 고객센터 카테고리

| 카테고리 | 일본어 라벨 (가설) |
|---|---|
| 결제·환불 | 決済・返金 |
| 응시 환경·기술 | 受験環境・技術 |
| 시험 콘텐츠 | 試験コンテンツ |
| 인증서·결과 | 証明書・結果 |
| 회원·계정 | 会員・アカウント |

> FAQ 게시판·1:1 문의 작성 폼 모두 이 5개 분류로 시작.

## 6. 마이페이지 정보 구조 (트리)

```
マイページ (/me)
├── ① 대시보드 — 진입 시 기본 화면
├── ② 受験する試験 (/me/tests)
│   └── 試験詳細 (/me/tests/:id)
├── ③ 結果 (/me/results)
│   └── 結果詳細 (/me/results/:id)
├── ④ 証明書 (/me/certificates)
├── ⑤ 決済履歴 (/me/payments)
├── ⑥ 返金 (/me/refunds)
├── ⑦ プロフィール (/me/profile)
│   └── 退会 (/me/withdraw)
└── ⑧ お問い合わせ履歴 (/me/inquiries)
```

### 좌측 사이드바 메뉴 그룹 (데스크탑)

| 그룹 | 항목 |
|---|---|
| **試験・結果** | 受験する試験 / 結果 / 証明書 |
| **決済** | 決済履歴 / 返金 |
| **アカウント** | プロフィール / 退会 |
| **サポート** | お問い合わせ履歴 |

> 모바일은 1단 리스트, 동일 그룹 헤더로 시각적 구분.
