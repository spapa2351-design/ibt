# 시스템 맵

## 한 줄

**Newin이 단독으로 만드는 3개의 웹 프론트엔드 + 1개의 AI 엔진 + 1개의 결제 모듈. 응시자용은 동일 웹 앱을 네이티브 쉘로 감싼다.**

## 이 한 줄을 지지하는 이유 5개

1. 메인 사이트(접수·결제), 관리자 백오피스, 응시자 클라이언트는 사용자 주체와 권한이 다르므로 각각 별도의 프론트엔드로 본다.
2. AI 엔진은 출제 보조·STT·채점·피드백 4가지 책임을 한 모듈에서 다룬다 — 일관된 평가 기준 학습이 핵심이라 분리 시 신뢰도가 흔들린다.
3. 결제는 외부 일본 PG 1개를 연동하는 단일 모듈이므로, 메인 사이트 안에 흡수되는 박스로 본다 (수익 모델 자체의 기획은 본 문서 범위 외).
4. **응시자 클라이언트는 단일 반응형 웹 앱**. 데스크탑은 브라우저 직접 접속, iOS/Android는 동일 웹 앱을 **네이티브 쉘(WebView 래퍼)**로 감싼 앱으로 배포. 모바일 웹은 폴백.
5. 마이크/녹음 같이 네이티브 권한·안정성이 핵심인 기능은 **Web API 1차 → 안정성 부족 시 네이티브 플러그인 fallback** 전략. 따라서 클라이언트 코드베이스는 1개로 유지된다.

## 응시자 클라이언트 — 단일 웹 앱 + 네이티브 쉘

**구조**: 코드베이스 1개 (반응형 웹 앱). 데스크탑은 브라우저로 접속. iOS/Android는 같은 웹 앱을 WebView 기반 네이티브 쉘로 감싸서 앱스토어 배포.

| 진입 디바이스 | 진입 방식 | 뷰포트 | 비고 |
|---|---|---|---|
| 데스크탑 (Win/Mac) | 브라우저 (Chrome/Safari/Edge) | **데스크탑 뷰포트** (2단 분할 — 좌: 문제, 우: 응시) | 사용자 주력. 마이크 권한·녹음 안정성 가장 높음 |
| iOS 앱 | App Store 다운로드 (네이티브 쉘) | **모바일 뷰포트** (1단 스택) | 사용자 주력. WebView 권한·녹음 안정성은 검증 대상 |
| Android 앱 | Play Store 다운로드 (네이티브 쉘) | **모바일 뷰포트** (1단 스택) | 사용자 주력 |
| 모바일 브라우저 | Safari/Chrome | **모바일 뷰포트** (1단 스택) | 폴백. PWA 형태로 설치 유도 가능 |

### 마이크·녹음 fallback 전략

1. **1차**: Web API (`MediaRecorder`, `getUserMedia`) — 데스크탑·앱·모바일 웹 모두 동일 코드
2. **2차 (필요 시)**: 네이티브 쉘 안에서만 네이티브 마이크 플러그인으로 우회
3. 검증 포인트: iOS WebView의 백그라운드 녹음 안정성, 권한 요청 UX, 인터럽션(전화·알람) 처리

> ⚠️ 본 디자인 자료는 모바일 1단 레이아웃만 포함. **반응형의 데스크탑 뷰포트(2단) 디자인은 부재** (`open-questions.md` Q13).

## 시스템별 책임 경계

### 메인 사이트 (Web)

| 영역 | 포함 |
|---|---|
| 회원 관리 | 회원가입(소셜 검토), 로그인, 계정 찾기, 비밀번호 재설정 |
| 마이페이지 | 구매 시험 관리, 응시 이력, 결제 내역, 회원정보 수정·탈퇴, **앱 오픈 진입점** |
| 시험 접수 | 시험 목록·필터, 시험 상세, 구매 |
| 결제 | 외부 일본 PG 1개 연동, 영수증 발행 |
| 인증서 | 성적 기반 인증서 PDF 발급, 위변조 확인 |
| 고객센터 | 공지·FAQ, 1:1 문의, 모든 페이지 하단 플로팅 [문의/FAQ] 버튼 |

### 관리자 백오피스 (Web)

| 영역 | 포함 |
|---|---|
| 시험 관리 | 시험 개요 등록, 공개 상태, 응시 조건/옵션, 응시 기간 |
| 문제 관리 | 문제 등록 (251020 ver.3 5영역 — 짧은 문장 읽기·질문 대답·생활 속 대화·그림 설명·화용 과제), AI 자동 생성 보조, 에디터 수정 |
| 시험지 관리 | N문항 단위 시험지 세트 구성, Drag & Drop 정렬, 배점 일괄 설정 |
| 응시자 관리 | 시험별 응시자 목록·결과·답안 초기화·결과 공개·CSV 다운로드 |
| 평가자 UI | 3단 분할 뷰, STT 싱크, 매크로 코멘트, 2인 교차 채점 |
| 회원/권한 | 회원 정보 조회, 관리자 등급별 권한 설정 |
| 통계 | 응시 통계, 문항별 정답률, AI 평가 품질 |
| 시스템 | 인증서 템플릿, 약관/개인정보처리방침, 2-factor, 활동 로그 |

### 응시자 클라이언트 (단일 반응형 웹 앱 — 데스크탑 브라우저 / iOS·Android 앱 / 모바일 웹)

| 영역 | 포함 |
|---|---|
| 응시 준비 | 진행 중 시험 목록, 응시 이력, 시험 안내, 환경 점검(마이크/볼륨/5초 리플레이/네트워크) |
| 오리엔테이션 | 튜토리얼 영상 또는 슬라이드 강제 시청, 주의사항 동의 |
| 응시 | (데스크탑) 좌:문제·우:응시 2단 / (모바일) 1단. 타이머(준비:파랑·녹음:빨강), 비프음, 자동 녹음, 메모장, 자동 저장·전송 |
| 결과 | 종합 결과(총점·등급), 5각 레이더 차트(발음·유창성·어휘·문법·내용 구성), 항목별 세부 점수·AI 코멘트 |
| 피드백 | 문항별 AI 피드백, STT 비교, 추천 연습 |
| 인증서 | PDF 다운로드 |
| 재응시 | 시스템·네트워크 에러 시 재응시 티켓 발행 (관리자 로그 확인 후) |

### AI 엔진

| 영역 | 포함 |
|---|---|
| 출제 보조 | 유형·주제·난이도·필수 문법 입력 → LLM이 ①지문 ②TTS 스크립트 ③모범 답안 ④채점 기준표 자동 생성 |
| STT | 응시자 녹음 → 텍스트 변환 |
| 채점 | 발음·유창성·어휘·문법·내용 구성 점수 산출 + 등급 매핑 |
| 피드백 생성 | 문항별 등급(A~F 학습용) + 답안 vs STT 비교 + 추천 연습 (자음/받침/억양) |

### 결제 모듈

| 영역 | 포함 |
|---|---|
| 외부 PG 연동 | 일본 PG 1개사 연동 예정 (사명 미확정) |
| 결제 수단 | 신용카드 + QR 결제 (PG가 지원하는 범위 내) |
| 영수증 | PDF 발행 |
| 환불 처리 | 운영 정책에 따른 환불 트랜잭션 (정책 본문은 본 문서 범위 외) |

## 책임 경계 — 누가 무엇을 책임지는가

| 시나리오 | 1차 책임 |
|---|---|
| 응시 중 마이크 권한 미허용 → 녹음 실패 | **응시자 (사전 점검 통과 약관 동의)** |
| 응시 중 네트워크 끊김 → 답안 유실 | **시스템 (자동 저장·재전송)** |
| AI 채점 결과 이의 제기 | **운영 1차 → 평가자 재검토 → AI 엔진 재학습 피드백** |
| 시스템 장애로 시험 진행 불가 | **시스템 (재응시 티켓 발행)** |
| 결제 실패·이중 결제 | **메인 사이트 결제 모듈 (트랜잭션 정합성) + 외부 PG** |
| 출제 기준 변경 | **외부 콘텐츠 입력 → AI 엔진 재학습** |

## 점선 영역 (미결정)

1. **네이티브 쉘 기술 스택**: Capacitor 가설이 가장 유력하지만 미확정. `open-questions.md` Q4-A.
2. **마이크/녹음 — Web API vs 네이티브 fallback 기준**: iOS WebView 검증 후 결정. `open-questions.md` Q4-B.
3. **반응형의 데스크탑 뷰포트 응시 UI 디자인**: 1순위 디바이스인데 본 디자인 자료에 부재. `open-questions.md` Q13.
4. **평가자 UI 런칭 시점**: 2차 채점 운영 시점에 종속. `open-questions.md` Q6.
