# Critical 4개 결정 안건 (Phase 1 차단)

> 본 문서는 **외부 결정 미팅용 1페이지 어젠다**다. 답이 채워지면 본 문서는 삭제하고 본문 4개에 분산 반영한다.

## 한 줄

**1차 오픈(2026.9) 디자인·개발 발주를 시작하기 위해 GAP-01 / GAP-02 / GAP-11 / GAP-12 4개 결정이 필요하다.**

## 이 한 줄을 지지하는 이유 5개

1. 4개 모두 *🔴 Critical* — 결정 없이 진행하면 *추측에 기반한 디자인·DB·스펙*이 박혀 나중 수정 비용이 일정 위험으로 직결된다.
2. 4개 중 3개는 외부(한국교육재단·외부 PG·디자인 발주처)에 결정 권한이 있으므로, 의사결정 리드 타임을 *이번 주*에 확보해야 한다.
3. 결정 1개당 영향 영역이 *디자인 + DB + UI 워크플로우 + 운영 정책* 4개에 걸쳐 있어, 각 결정이 미루어질수록 차단 영역이 기하급수적으로 늘어난다.
4. Important / Minor 갭은 가정으로 1차 진행이 가능하지만 Critical 4개는 가정 자체가 위험하다.
5. 본 안건이 결정되면 라운드 2 진입(`07-grading.md` / `08-feature-spec.md` 깊이 확장)이 즉시 가능하다.

## GAP-01 — 등급 체계 확정

### 배경

자료 4종 사이에 등급 표기가 일관되지 않다.

| 자료 | 표기 |
|---|---|
| 마스터 문서 | "A~D 4단계 진단 등급" |
| 시험 결과 디자인 (`img/시험(채점) 결과.png`) | **1급 / 6급** |
| AI 피드백 디자인 (`img/1번 문항- Ai 피드백.png`) | **B / Good** (A~F 추정) |
| 응시자 관리 백오피스 | "1급" 표시, "등급별" 필터 |

### 옵션

| # | 옵션 | 장점 | 단점 |
|---|---|---|---|
| **A (권장)** | **2-track 분리** — 종합 진단 = 1~6급(TOPIK 매핑) / 문항별 학습용 = A~F | 디자인 자료 3종을 모두 수용 / 응시자가 TOPIK 등급으로 자기 위치를 즉시 이해 | DB·UI에 두 트랙을 모두 모델링해야 함 |
| B | A~D 4단계 단일 (마스터 문서 그대로) | 단순 / 디자인 충돌 적음 | 디자인 자료의 1급/6급·B/Good 모두 폐기 |
| C | 1~6급 단일 (디자인 그대로) | 디자인 자료 보존 | 마스터 문서의 A~D·문항별 A~F 폐기, 학습용 피드백 등급 표현 부재 |

### 결정 후 영향

- **디자인**: 성적표 화면(5각 차트 색상·등급 배지) / AI 피드백 모달(등급 박스 색상)
- **DB**: `attempt.overall_grade` 컬럼 enum / `feedback_per_question.grade` 별도 컬럼
- **AI 출력 스펙**: 1차 채점 결과 JSON에 두 등급 동시 포함
- **시험 통과 조건 정의**: 통과 = 종합 N급 이상 / 평균 X점 이상

### 결정 주체

**한국교육재단** (출제·평가 기준 담당)

### 마감

⛳ **2주 내** — 디자인 시스템(`08-design-system.md`)의 등급 배지 색상 매핑 확정에 직접 영향.

## GAP-11 — 데스크탑 응시 UI 디자인 발주

### 배경

**데스크탑이 1순위 디바이스**로 명시됐으나, 본 디자인 자료의 응시 화면 13장은 **모두 모바일 1단**.

마스터 문서: *"좌측(문제 영역) / 우측(응시 영역) 2단 분할"*

### 옵션

| # | 옵션 | 장점 | 단점 |
|---|---|---|---|
| **A (권장)** | **반응형 데스크탑 뷰포트 디자인 추가 발주** (모바일 디자인을 데스크탑 break로 확장) | 디자인 일관성 유지 / 1차 오픈 일정 수렴 | 디자인 비용 추가 |
| B | 모바일 디자인 그대로 데스크탑에서 *최대 768px 폭 제한* | 발주 비용 0 | 1순위 디바이스에서 화면 활용도 낮아 사용자 경험 저하 |
| C | 데스크탑은 라운드 2로 미루고, 1차 오픈은 모바일·앱만 운영 | 발주 비용·일정 절감 | 사용자 1순위 디바이스 미지원이라 시장 출시 의미 약화 |

### 결정 후 영향

- **디자인 발주**: 응시 화면 13장 + 환경 점검 + 오리엔테이션 + 결과 = **약 20+ 화면 데스크탑 뷰포트 추가 발주**
- **개발**: 반응형 break 추가 코딩
- **QA**: 데스크탑 테스트 케이스 추가

### 결정 주체

**Newin 내부** (디자인 발주 의사결정)

### 마감

⛳ **즉시** — 디자인 발주 리드 타임이 가장 길다.

## GAP-12 — 결제 PG사 선정

### 배경

자료: *"일본 PG사 1개사 연동 예정"* — 사명·계약 미확정.

### 옵션 (시장 상위 후보)

| # | PG사 | 장점 | 단점 |
|---|---|---|---|
| A | **GMO Payment Gateway** | 일본 시장 점유율 1위 / TOPIK 일본 사이트도 사용 (응시자 익숙) / 신용카드·PayPay·コンビニ 다수 지원 | 가격 협상력 약함 |
| B | **SBペイメント (SB Payment Service)** | SoftBank 계열, PayPay 직접 통합 강함 | 인지도 GMO 대비 약간 낮음 |
| C | **Stripe Japan** | 글로벌 표준 SDK / 개발자 친화적 | 일본 결제 수단 (コンビニ·은행 송금) 일부 미지원 |
| D | **Komoju / Pay.JP** | 일본 결제 특화 | 시장 점유율 낮음 / 안정성 검증 적음 |

> **권장**: **GMO Payment Gateway** — 마스터 문서가 이미 "GMO-PG 연동" 가설로 작성되어 있고, 현 TOPIK 일본 사이트와도 호환.

### 결정 후 영향

- **디자인**: 결제 페이지 (PG SDK 호출 흐름) / 영수증 PDF 양식
- **개발**: PG SDK 선정 / 결제 트랜잭션 정합성 / 환불 API
- **운영**: 결제 수단 매트릭스 (신용카드·PayPay·컨비니·가상계좌 중 선택)
- **법무**: 일본 적격청구서(인보이스) 양식 PG 호환성 검토

### 결정 주체

**외부 협상** (BLOCKED)

### 마감

⛳ **4주 내** — 결제 모듈 구현이 1차 오픈의 *결제 → 응시* 핵심 동선을 차단.
