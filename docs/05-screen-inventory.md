# 화면 인벤토리 + 반응형 매트릭스 + 발주 우선순위

## 한 줄

**본 자료의 27개 화면은 응시자 모바일 1단에 편중되어 있고, 1차 오픈 차단 화면 두 영역(데스크탑 응시 2단·메인 사이트 거의 전체)이 비어 있다.**

## 이 한 줄을 지지하는 이유 5개

1. 사용자 1순위 디바이스인 **데스크탑 응시 2단 뷰포트** 디자인이 부재(GAP-11 🔴 Critical) — 응시 클라이언트 개발이 시작될 수 없다.
2. **메인 사이트(서비스 웹사이트)**는 가입·로그인·시험 목록·시험 상세·결제·마이페이지 *어느 한 화면도* 자료에 없다.
3. 본 자료의 응시자 화면 13장은 모두 **모바일 1단 레이아웃**으로 그려져 있어 반응형의 출발점은 잡혔지만 데스크탑 매핑이 미정.
4. 관리자 화면 7장은 백오피스 핵심 메뉴(시험·문제·시험지·응시자·시험별 응시자 상세)를 커버하지만, 시험 통과 조건 설정·환경 점검 시각화·평가자 UI 등이 비어 있다.
5. 발주 우선순위를 *1차 오픈 차단 → 1차 오픈 핵심 → 라운드 2 이후* 3단계로 분리하면 디자인 일정과 개발 발주가 동시 진행 가능하다.

## 3. 반응형 뷰포트 매트릭스

| 영역 | **데스크탑** (≥1024px) | **태블릿** (768~1023) | **모바일·앱** (≤767) |
|---|---|---|---|
| 메인 사이트 (랜딩·시험 목록) | 표준 멀티 컬럼 (3~4열 카드) | 2열 카드 | 1열 스택 |

## 6. 관리자 UI 작성 현황

좌측 사이드바 5블록 — **대시보드 → 응시자 → 회원 → 시험 → 설정** (회원·응시자 그룹이 최상단, 운영팀 일상 사용 빈도 기준).

| 파일 | 내용 |
|---|---|
| `ui-admin-dashboard.html` | 대시보드 (4 KPI · 미답변 문의 · 빠른 액션 · 7일 응시 추이) |
| `ui-admin-members.html` | 응시자 회원 목록 |
| `ui-admin-payments.html` | 결제 관리 (5상태 + 환불 처리) |
| `ui-admin-question-bank.html` | 문제 은행 (분류 트리 + 표) |
| `ui-admin-question-create.html` | 문제 만들기 — 251020 ver.3 5영역 탭 (짧은 문장 읽기·질문 대답·생활 속 대화·그림 설명·화용 과제), 영역별 폼 동적 변경, **응시자 폰 mock 라이브 프리뷰** 우측 sticky 사이드바 |
| 시험 목록 / 시험 등록 / 응시 결과 / 관리자 계정 / 인증서 템플릿 / 공지·FAQ / 약관 / 보안 | 라운드 2 |

KoPASS admin은 **말하기 전용**. 객관식·주관식은 본가 시스템(별도 인프라). 문제 만들기는 251020 ver.3에 정의된 5영역(영역 6은 ver.3에서 삭제)을 탭으로 분기하고 영역별 한국어/일본어 지시문·시간·배점이 자동 적용됨. 미리보기는 모바일 동등 비중을 반영한 **핸드폰 frame** mock으로 구성, 영역별 콘텐츠(텍스트 문장 / 음성 펄스 + 질문 / 일러스트 + 말풍선 / 큰 이미지 / 일본어 상황 박스)가 분기 표시됨.

폰트는 Pretendard(한·일 동시 지원, `Pretendard Variable` + `Pretendard JP Variable`).
