## 1. 개요

### 1.1 목적
MaroPass portal(kopass.*)의 페이지 계층·네비게이션·분류체계를 정의하여 이후 화면 설계(Wireframe/Hi-Fi)와 라우팅 구현의 공통 기준을 마련한다. 화면별 상세 인터랙션은 기능명세서에서 다루며, 본 문서는 "무엇이 어디에 놓이는가"까지를 범위로 한다.

### 1.2 범위
- 대상: portal(신규 서비스 사이트). 회원가입/로그인, 시험 탐색·구매, 마이페이지, 고객센터, 정책 페이지 일체.
- portal은 데스크탑 + 모바일 웹만 대응. 네이티브 앱은 라운드2.
- 사용자 web(응시 화면)·관리자 web은 기존 터치클래스 자산을 재사용하므로 본 문서에서는 진입/복귀 연결점만 요약 언급한다. 내부 IA는 각 시스템 기존 문서 참조.

### 1.3 전제
- 기획 착수 2026년 5월 초, 테스트 납품 11월 초, 정식 오픈 12월 초. Phase1 판매 상품은 '말하기' 진단평가 단일 영역.
- 응시는 portal에서 시작하지 않는다. 마이페이지 '응시할 시험'에서 새 창(window.open)으로 사용자 web을 띄운다(정책10). portal은 결제·결과열람·인증서 다운로드까지 담당.
- 로그인 수단은 이메일 + 소셜 3종(LINE / Google / Apple).
- UI 언어는 일본어 1차. 본 문서의 일본어 라벨은 전부 가설이며 네이티브 검수 후 확정한다.

## 2. IA 설계 원칙

1. 단일 핵심동선 우선 — "시험 찾기 → 상세 → 구매 → 마이페이지에서 응시"를 최상위 동선으로 두고, 그 외 기능은 이 흐름을 방해하지 않는 위치에 배치.
2. 2클릭 룰 — 홈 기준 주요 목적지(시험 목록, 마이페이지, 고객센터)는 2클릭 이내 도달. 마이페이지 하위 항목은 사이드바로 1뎁스 평탄화.
3. 회원영역 셋오프 — 로그인·회원가입·프로필은 헤더 우상단에 모아 두고 핵심 콘텐츠 네비(GNB 3메뉴)와 분리. 인지 부담 축소 목적.
4. 디바이스별 네비 분리 — 데스크탑은 상단 GNB, 모바일은 하단 탭 4개. 하단 탭에는 회원 진입(가입/로그인)을 넣지 않고 우상단으로 뺀다.
5. 라벨 일본어 1차 / 콘텐츠 한국어 — 화면 라벨은 일본어, 시험 문항·지문 등 학습 콘텐츠 영역만 한국어(:lang(ko)). 언어 혼용 지점은 라벨 규칙(8장)에서 관리.

원칙 간 충돌 시 1 > 2 순으로 우선. (예: 부가 기능을 2클릭에 넣기 위해 핵심동선을 흐리는 배치는 지양.)

## 3. 사이트맵

L0 = 사이트 루트, L1 = GNB/주요 섹션, L2 = 섹션 하위, L3 = 상세/개별.

```
MaroPass portal (kopass.*)  [L0]
│
├─ 홈 /                                        [L1]
│
├─ 시험 /tests                                  [L1]
│   ├─ 시험 목록(레벨·영역 필터)                 [L2]
│   └─ 시험 상세 /tests/:slug                    [L3]
│        └─ (구매 클릭 → 회원 게이트 → 결제)
│
├─ 결제 /checkout/:cartId                        [L1] * 진입 한정, GNB 비노출
│   ├─ 결제 완료 /checkout/:cartId/complete       [L2]
│   └─ 결제 실패 /checkout/:cartId/fail            [L2]
│
├─ 마이페이지 /me                                [L1] * 로그인 필요
│   ├─ 대시보드 /me                              [L2]
│   ├─ 응시할 시험 /me/tests                       [L2] → (새 창) 사용자 web
│   ├─ 결과 /me/results  ├ 결과 상세 /me/results/:id [L2/L3]
│   ├─ 인증서 /me/certificates                    [L2]  (PDF 다운로드)
│   ├─ 결제내역 /me/payments                       [L2]
│   ├─ 환불 /me/refunds                           [L2]
│   ├─ 회원정보 /me/profile                        [L2]
│   ├─ 회원탈퇴 /me/withdraw                       [L2]
│   └─ 1:1 문의내역 /me/inquiries                   [L2]
│
├─ 고객센터 /support                             [L1]
│   ├─ 공지 /support/notice                       [L2]
│   ├─ FAQ /support/faq                          [L2]
│   └─ 1:1 문의 /support/inquiry                   [L2]  (작성; 로그인 필요)
│
├─ 계정 (헤더 우상단 셋오프)                       [L1]
│   ├─ 로그인 /signin                             [L2]
│   ├─ 회원가입 /signup                           [L2]
│   ├─ 비밀번호 재설정 /reset-password              [L3]
│   └─ 계정 찾기 /find-account                     [L3]
│
└─ 정책/회사 (푸터)                               [L1]
    ├─ 이용약관 /terms                            [L2]
    ├─ 개인정보처리방침 /privacy                    [L2]
    ├─ 특정상거래법 표시 /legal/tokushoho           [L2]
    └─ 회사정보 /company                          [L2]
```

메모: 결제 섹션은 상세 페이지에서만 진입하는 흐름상 노드로, GNB에는 노출하지 않는다. 사이트맵에는 계층 표현을 위해 L1로 기재.

## 4. 글로벌 네비게이션

### 4.1 데스크탑 GNB
로고 좌측, 핵심 3메뉴 중앙, 회원 셋오프 우측의 3분할 구조.

```
┌──────────────────────────────────────────────────────────────┐
│  [MaroPass]      試験一覧   マイページ   サポート      ログイン │
│   (로고/홈)                                        [프로필 아바타]│
└──────────────────────────────────────────────────────────────┘
```

| 위치 | 항목 | 라벨(일본어 가설) | 링크 | 비고 |
|------|------|-------------------|------|------|
| 좌 | 로고 | MaroPass | `/` | 클릭 시 홈 |
| 중앙 | 시험 목록 | 試験一覧 | `/tests` | |
| 중앙 | 마이페이지 | マイページ | `/me` | 비로그인 클릭 시 `/signin` |
| 중앙 | 고객센터 | サポート | `/support` | |
| 우 | 로그인 | ログイン | `/signin` | 비로그인 시 노출 |
| 우 | 프로필 | (아바타) | `/me` | 로그인 시 로그인 버튼 대체, 드롭다운(마이/로그아웃) |

- 마이페이지는 로그인 상태에서만 실사용. 비로그인 클릭은 게이트(정책9)로 로그인 유도.
- 언어 토글(日/한)은 현 Hi-Fi 시안 한정 우상단 배치. 정식 오픈 시 노출 여부 TBD.

### 4.2 모바일 하단 탭
탭 4개. 회원가입/로그인은 탭에서 제외하고 화면 우상단에 배치.

```
┌──────────────────────────────────────┐
│                (콘텐츠)               │
├──────────────────────────────────────┤
│  ホーム    試験    マイ    サポート    │
│  (Home) (ListChecks)(User)(HelpCircle) │
└──────────────────────────────────────┘
```

| 탭 | 라벨(일본어 가설) | 링크 | 비고 |
|----|-------------------|------|------|
| 홈 | ホーム | `/` | |
| 시험 | 試験 | `/tests` | |
| 마이 | マイ | `/me` | 비로그인 시 `/signin` |
| 사포트 | サポート | `/support` | |

아이콘은 예시. 최종 픽토그램은 디자인 발주 시 확정.

### 4.3 푸터
전 페이지 공통. 좌측 회사/법적 링크, 우측 SNS. 우하단 문의 플로팅 버튼 상시 노출.

- 법적/정책: 이용약관, 개인정보처리방침, 특정상거래법 표시, 회사정보
- SNS: LINE 공식계정, X, (인스타 검토 중)
- 플로팅: 문의 버튼(→ `/support/inquiry`, 비로그인 시 게이트)

특정상거래법(特定商取引法) 표시는 일본 전자상거래 필수 고지. 오픈 전 법무 확인 필요.

## 5. 페이지 목록 · 라우팅

라벨은 일본어 가설 / 한국어는 내부 커뮤니케이션용. depth는 3장 기준.

| `/signup` | 会員登録 | 회원가입 | L2 | - | |
| `/reset-password` | パスワード再設定 | 비밀번호 재설정 | L3 | - | |
| `/find-account` | アカウント検索 | 계정 찾기 | L3 | - | 이메일 기준 |

## 6. 마이페이지 정보구조

로그인 후 진입. 좌측 사이드바(모바일은 상단 접이식/셀렉트) 4개 그룹으로 구성. 진입 기본 화면은 대시보드.

```
/me (대시보드)
├─ [시험·결과]
│    ├─ 응시할 시험      /me/tests
│    ├─ 결과            /me/results
│    └─ 인증서          /me/certificates
├─ [결제]
│    ├─ 결제내역        /me/payments
│    └─ 환불            /me/refunds
├─ [계정]
│    ├─ 회원정보        /me/profile
│    └─ 회원탈퇴        /me/withdraw
└─ [고객센터]
     └─ 문의내역        /me/inquiries
```

| 그룹 | 라벨(일본어 가설) | 포함 항목 | 비고 |
|------|-------------------|-----------|------|
| 시험·결과 | 試験・結果 | 응시할 시험, 결과, 인증서 | 대시보드 상단 요약과 연동 |
| 결제 | お支払い | 결제내역, 환불 | 환불은 결제내역에서도 진입 가능 |
| 계정 | アカウント | 회원정보, 회원탈퇴 | 탈퇴는 그룹 최하단 배치 |
| 고객센터 | サポート | 문의내역 | 신규 문의는 `/support/inquiry` |

대시보드에는 응시 예정 시험, 최근 결과, 미확인 인증서 등을 카드로 요약. 카드 구성 항목은 기능명세에서 확정.

## 7. 분류체계 (Taxonomy)

### 7.1 시험 카테고리
Phase1은 '말하기' 단일 영역. 필터 축은 레벨 × 영역 2종으로 두되 영역은 현재 단일값.

| 축 | 값 | 대응 | 비고 |
|----|----|------|------|
| 레벨 | 초급 | TOPIK I 수준 | |
| 레벨 | 중상급 | TOPIK II 수준 | |
| 영역 | 말하기 | Speaking | Phase1 단독 |
| 영역 | (쓰기·듣기·읽기) | - | 라운드2. 목록 필터에 자리만 확보 |

등급(1~6급) 표기 확정 전. 레벨 2분류와 등급 체계의 매핑은 미결정(9장).

### 7.2 고객센터 카테고리
FAQ·문의 공통 분류 5종.

| 코드 | 분류 | 라벨(일본어 가설) |
|------|------|-------------------|
| pay | 결제·환불 | お支払い・返金 |
| tech | 응시환경·기술 | 受験環境・技術 |
| content | 시험 콘텐츠 | 試験コンテンツ |
| cert | 인증서·결과 | 認定証・結果 |
| account | 회원·계정 | 会員・アカウント |

문의 작성 폼의 분류 셀렉트도 위 5종을 그대로 사용. 코드값은 내부 식별용.

## 8. 라벨 · 표기 규칙

### 8.1 원칙
- 화면 UI 라벨은 일본어 1차. 버튼·메뉴·안내문 전부 해당.
- 시험 문항, 지문, 예시 답안 등 한국어 학습 콘텐츠 영역만 한국어로 두고 `:lang(ko)` 지정(폰트·자간 별도 처리).
- 한국어/일본어가 한 화면에 혼재하는 지점(예: 시험 상세의 설명은 일본어, 샘플 문항은 한국어)은 시각적으로 영역을 구분.
- 숫자·통화는 엔화(¥) 기준 표기.

### 8.2 주요 라벨 대조표 (발췌)
일본어는 전부 가설. 네이티브 검수 후 확정한다.
