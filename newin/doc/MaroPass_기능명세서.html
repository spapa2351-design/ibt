1. 개요

1.1 목적

MaroPass IBT는 일본인 한국어 학습자를 대상으로 하는 '말하기' 진단평가 서비스임. TOPIK을 대체하지 않고 보완·진단하는 포지션으로, 말하기 5개 영역을 IBT(인터넷 기반 시험) 방식으로 응시·채점·결과 제공함. 브랜드는 MaroPass(도메인 kopass.*), 운영사는 뉴인재팬.

본 문서는 세 개 표면 중 신규 개발 비중이 가장 큰 MaroPass portal의 기능을 중심으로 기술함. 실제 응시가 이루어지는 터치클래스 사용자 web, 운영이 이루어지는 터치클래스 관리자 web은 기존 자산 재사용을 전제로 하되, MaroPass 특화 신규 영역과 portal에서 넘어오는 연계 흐름만 이 문서에서 다룸.

일정은 기획 착수 2026년 5월 초, 테스트 납품 11월 초, 정식 오픈 12월 초. 1차 오픈은 데스크탑·모바일 웹만 지원하며, 네이티브 앱은 라운드 2 이후.

1.2 범위

포함:

  • MaroPass portal 전 기능 (가입·로그인, 시험 목록/상세, 결제, 마이페이지, 결과 진입 링크, 고객센터, 알림)
  • portal → 사용자 web 응시 진입(새 창, SSO) 연계
  • 사용자 web 응시·본인확인·환경점검·결과 확인의 연계 기술 (상세 구현은 사용자 web 팀 소관)
  • 관리자 web MaroPass 신규 메뉴 5개 및 문제 만들기 말하기 5영역 탭의 연계 기술

제외(별도 문서 또는 후속 라운드):

  • 부정행위 방지, 관리자 등급별 권한, 데이터 보존/삭제 정책 (라운드 3, 12-security.md/13-anti-cheat.md)
  • 평가자 2차 채점 UI 및 워크플로우 (라운드 2)
  • 성능·보안 상세 요구 (라운드 3에서 별도 명세)

1.3 용어

산문 중 반복 사용하는 약어를 아래에 정리함. 전체 용어는 용어집(01-glossary.md) 참조.

용어 설명
portal MaroPass portal. 신규 개발되는 브랜드·결제·인증서 표면 (kopass.*)
사용자 web 기존 터치클래스 사용자 web. 본인확인·환경점검·응시·결과가 이루어지는 학습 자산 표면
IBT Internet-Based Test. 인터넷 기반 응시 방식
PG Payment Gateway. 결제 대행. 1차 오픈은 일본 PG(신용카드+QR), 사업자 미확정(TBD)
STT Speech-to-Text. 응시자 음성 답안을 텍스트로 변환. 결과 화면에서 답안과 대조 노출
SSO Single Sign-On. portal 로그인 세션을 사용자 web에 토큰으로 전달해 자동 로그인
응시 윈도우 결제 후 응시 가능한 기간. 1차 오픈 14일, 기간 내 24시간 언제나 응시 가능 (정책8)

4. 기능 상세 명세

우선순위 상 기능을 중심으로 깊이 있게 기술하고, 부수 기능은 요지만 정리함. 흐름 스텝의 버튼·문구는 일본어 UI 기준으로 예시 표기(일본어 라벨 네이티브 검수 TBD).

4.1 FS-MEM-01 / 이메일 회원가입

개요 비회원이 이메일로 계정을 생성하는 기능. 무료 샘플·구매 진입 시 회원 게이트(정책9)를 통해 진입하는 경로가 주 유입이므로, 가입 화면 상단에 진입 의도(intent) 배지를 노출해 이탈을 줄임. 수집 개인정보는 최소화함.

선행조건

  • 미로그인 상태
  • 유효한 수신 가능 이메일 보유

기본 흐름

  1. 시험 상세에서 "無料サンプル" 또는 "購入する" 클릭, 또는 상단 "会員登録" 진입
  2. 이메일·비밀번호 입력
  3. 약관 동의: 필수(이용약관·개인정보 취급) 체크, 선택(마케팅 수신) 체크
  4. "認証メール送信" 클릭 → 인증 메일 발송
  5. 메일 내 인증 링크 클릭 → 이메일 인증 완료
  6. 최소 개인정보(이름 등) 입력 후 가입 완료
  7. intent 값에 따라 원래 동작으로 복귀 (FS-MEM-06)

예외·대체 흐름

  • 이미 가입된 이메일: "既に登録済み" 안내 후 로그인 유도
  • 인증 링크 만료(유효시간 TBD): 재발송 버튼 제공
  • 소셜 계정과 동일 이메일 존재: 계정 통합 정책 미정(TBD, 라운드2에서 결정 가능)

관련 화면: 회원가입, 약관 동의, 이메일 인증 완료

데이터·입력: 이메일, 비밀번호, 이름, 약관 동의 항목별 동의 여부·시각, intent, return

정책·규칙: 정책9(회원 게이트). 약관 동의 시각은 CS·클레임 대응 근거로 저장.

비고: 수집 항목 최종 확정은 개인정보처리방침 확정과 연동. 인증 링크 유효시간 TBD.

4.2 FS-MEM-02 / 소셜 회원가입·로그인

개요 LINE·Google·Apple 소셜 로그인 제공. 일본 시장 특성상 LINE 비중이 높을 것으로 보고 1순위로 배치함. 최초 연동 시 약관 동의를 반드시 경유함.

기본 흐름

  1. 로그인/가입 화면에서 "LINEで続ける" 등 선택
  2. 각 provider 인증 화면으로 이동·동의
  3. 콜백 복귀 후, 신규 사용자면 약관 동의 화면 → 가입 완료 / 기존 사용자면 로그인 완료
  4. intent 있으면 원래 동작 복귀

예외·대체 흐름

  • provider 인증 취소·실패: 로그인 화면으로 복귀, 오류 안내
  • 이메일 미제공 provider(Apple 비공개 릴레이 등): 릴레이 이메일로 계정 생성. 알림 발송 가능 여부 확인 필요(TBD)

정책·규칙: 정책9. provider 앱 등록·심사(특히 LINE·Apple)는 오픈 전 완료 필요 — 일정은 WBS 참조.

비고: 각 provider 개발자 계정·리다이렉트 URI 등록은 개발 착수 전 선행.

4.3 FS-MEM-06 / 회원 게이트 (intent 복귀)

개요 무료 샘플 응시·구매는 로그인 회원만 가능함(정책9). 비로그인 사용자가 해당 액션을 클릭하면 원래 의도를 파라미터로 실어 가입/로그인으로 유도하고, 인증 완료 후 그 의도로 자동 복귀시킴. 시험 상세 열람과 샘플 미리듣기는 이 게이트의 예외로 비로그인 허용.

선행조건: 사용자가 샘플 또는 구매 액션을 트리거

기본 흐름

  1. "無料サンプル"(intent=sample) 또는 "購入する"(intent=purchase) 클릭
  2. 로그인 상태 확인
  3. 비로그인 시 ui-auth.html?intent={sample|purchase}&return={원래 URL 또는 마커}로 이동
  4. 가입/로그인 화면 상단에 intent 배지 노출 (예: 무료 체험 안내 / 결제 안내 문구)
  5. 인증 완료
  6. intent=sample → 시험 상세로 복귀 후 환경 점검(샘플용) 자동 발화
  7. intent=purchase → 결제 페이지로 이동

예외·대체 흐름

  • 이미 로그인 상태에서 액션 클릭: 게이트 미발동, 곧바로 해당 동작 진행
  • return 파라미터 유실·변조: 마이페이지 홈으로 안전 복귀

관련 화면: 시험 상세, 로그인/가입

데이터·입력: intent(sample|purchase), return

정책·규칙: 정책9. 본 게이트는 UI 레벨 장치이며, 실제 접근 통제는 API 인증 토큰 검사로 별도 보장. ※ 04-policy 정책9 참조.

비고: 배지 문구·이모지 사용 여부는 디자인/카피 검수에서 확정.

4.4 FS-EXM-02 / 시험 상세

개요 시험 한 건의 소개와 구매 진입을 담당하는 화면. 좌측에 콘텐츠(개요·구성·평가 영역·샘플 안내), 우측에 구매 사이드바(가격·응시기간 14일·응시시간·응시 환경 요건)를 배치함. 데스크탑은 우측 sticky 사이드바, 모바일은 하단 sticky CTA로 반응형 전환.

선행조건: 없음 (비로그인 열람 허용)

기본 흐름

  1. 시험 목록 또는 랜딩에서 특정 시험 카드 선택
  2. 좌측 콘텐츠·우측 사이드바 노출
  3. "無料サンプル(3問)" 또는 "購入する" 클릭 → 회원 게이트(FS-MEM-06) 경유

예외·대체 흐름

  • 판매 종료·비공개 시험: 상세 접근 시 안내 후 목록 복귀

관련 화면: 시험 상세 (데스크탑 2단 / 모바일 1단 + 하단 CTA)

데이터·입력: 시험ID, 레벨(초급/중상급), 가격, 응시기간, 응시시간, 환경 요건

정책·규칙: 정책9 예외(상세 열람·미리듣기 비로그인 허용)

비고: 중상급 말하기 유형 확정 전까지 중상급 상세 콘텐츠는 플레이스홀더로 노출(TBD).

4.5 FS-PAY-01~03 / 결제

개요 일본 PG를 통한 신용카드·QR 결제. 결제 완료 시 해당 시험에 대한 응시 윈도우(14일)를 부여하고, 기간 내 24시간 언제나 응시 가능하도록 티켓을 발급함(정책8). PG 사업자가 미확정이므로 결제 요청/콜백/영수증 항목의 세부 규격은 사업자 확정 후 보완함.

선행조건

  • 로그인 회원 (게이트 통과)
  • 결제 대상 시험 선택 완료

기본 흐름

  1. 시험 상세 → "購入する" → 결제 페이지
  2. 결제 수단 선택 (신용카드 / QR)
  3. PG 결제창 진입·인증·승인
  4. 콜백 수신 → 결제 완료 처리
  5. 응시 윈도우(14일) 부여, 마이페이지 "응시할 시험"에 등록
  6. 결제 완료 이메일 발송 (FS-NOTI-01)

예외·대체 흐름

  • 결제 실패: 실패 사유 안내, 재시도 유도. 티켓 미발급
  • 이중 결제·중복 승인: 결제 모듈에서 자동 회수 (PG 트랜잭션 정합성, 정책7)
  • 콜백 유실: 운영자가 결제 관리(FS-ADM-02)에서 상태 확인·수동 정합. ※ 04-policy 정책7 참조

관련 화면: 결제 페이지, 결제 완료/실패

데이터·입력: 주문ID, 시험ID, 결제수단, 금액, PG 거래ID, 승인시각

정책·규칙: 정책8(14일 윈도우), 정책7(환불·회수)

비고: PG 사업자 TBD. 확정 후 결제 요청·콜백·영수증·부분취소 규격 보완. 특정상거래법·적격청구서(인보이스) 표기 요건은 영수증 항목과 함께 정리.

4.6 FS-TST-01 / 응시 진입 (새 창 + SSO)

개요 응시는 portal이 아니라 사용자 web에서 이루어짐. 마이페이지 "受験する"(응시하기) 클릭 시 window.open()으로 사용자 web을 새 창으로 열고, portal 로그인 세션을 SSO 토큰으로 전달해 자동 로그인시킴. 새 창은 사용자에게 노출되는 UI 수단일 뿐이며, 본질은 브랜드 자산(portal)과 학습 자산(사용자 web)의 소유권 분리임(정책10). 팝업 차단에 대한 안내·재시도 가이드가 필수.

선행조건

  • 로그인 상태
  • 응시 윈도우가 유효한(만료 전) 구매 시험 보유

기본 흐름

  1. 마이페이지 "응시할 시험"에서 대상 시험의 "受験する" 클릭
  2. 안내 문구 노출: "受験は新しいウィンドウで行います。ポップアップがブロックされた場合は許可してください。"
  3. window.open()으로 사용자 web home 오픈
  4. 새 창이 SSO 토큰으로 사용자 web 자동 로그인
  5. 사용자 web에서 본인 확인(FS-TST-02)으로 진행

예외·대체 흐름

  • 팝업 차단 감지: 차단 안내 + "受験画面を開く" 재시도 버튼 노출
  • SSO 토큰 만료·검증 실패: 사용자 web에서 재로그인 유도 (portal 세션 확인)
  • 응시 윈도우 만료: "受験する" 비활성, 만료 안내 (환불·재구매 동선은 정책7·정책8 범위)

관련 화면: 마이페이지(portal), 사용자 web home

데이터·입력: 시험ID, 응시 인스턴스ID, SSO 토큰

정책·규칙: 정책10(새 창·소유권 분리), 정책8(윈도우). ※ 04-policy 정책10 참조

비고: SSO 토큰 규격·유효시간은 개발 리드 검토 후 확정(TBD). 사용자 web 결과 화면 하단에 "証明書・結果PDFはMaroPassマイページから" 안내 링크 포함.

4.7 FS-TST-03~06 / 사용자 web 응시 (연계)

개요 본 흐름은 사용자 web(터치클래스) 소관이며, portal은 진입만 담당함. 응시자는 시작·정지·건너뛰기 버튼을 갖지 않고, 시간이 흐르면 자동으로 다음 단계로 이동함(정책2). 아래는 연계 이해를 위한 요지 정리.

환경 점검 게이트 (FS-TST-03, 정책1)

  • 점검 항목: 마이크 권한 / 볼륨(데시벨 미터 시각화) / 5초 녹음 후 즉시 재생 / 네트워크
  • 통과 조건: 4개 항목 통과 + "통과 후 개인 기기 문제는 사용자 책임" 약관 동의
  • 동일 기기 30일 캐시로 재점검 부담 완화
  • 미통과 시 응시 진입 차단, 막힌 항목 명시

응시 클라이언트 (FS-TST-04, 정책2)

  • 말하기 5영역: 짧은 문장 읽기 / 질문 대답 / 생활 속 대화 / 그림 설명 / 화용 과제
  • 레이아웃: 데스크탑 2단(좌 문제 / 우 응시), 모바일 1단
  • 지시문 자동 표시·음성 자동 재생 → 준비 카운트다운(파란 타이머) → 비프음 → 녹음(빨간 타이머 + "녹음 중" + Waveform) → 자동 저장 → 다음 문항 자동 이동
  • 응시자 조작 버튼 없음

자동 저장·재전송 (FS-TST-05, 정책3)

  • 문항별 녹음 종료 직후 단말 로컬(IndexedDB) 저장
  • 네트워크 복구 시 자동 재전송, 응시자 조작 불요
  • 오프라인 표시는 하되 응시 중단 강요하지 않음

시험 자동 종료 (FS-TST-06, 정책4)

  • 마지막 문항 녹음 시간 종료 시 자동 일괄 전송 → 응시 종료 화면 → AI 채점 큐 등록
  • 중도 종료 버튼 없음
  • 끝까지 전송 실패 시 시스템 장애 재응시 티켓 흐름으로 흡수(정책6)

비고: 데스크탑 응시 2단 뷰포트 디자인은 미발주 상태(GAP-11, Critical). ※ 05-screen-inventory 참조. 이어서 응시 허용 여부는 1차 오픈 미허용 가능(정책4, 결정 필요).

4.8 FS-RST-01~04 / 결과 확인

개요 AI 1차 채점은 시험 종료 즉시 자동 진행되나, 응시자 노출은 운영자가 관리자 web에서 "결과 공개"를 클릭한 뒤에 이루어짐(정책5). 결과 확인 화면은 사용자 web에 있고, portal 마이페이지에는 진입 링크와 인증서·결과지 다운로드가 위치함.

선행조건

  • 응시 완료
  • 운영자의 결과 공개 완료 (FS-ADM-04)

기본 흐름

  1. 결과 공개 시 응시자에게 결과 공개 이메일 발송 (FS-NOTI-01)
  2. 마이페이지 또는 사용자 web에서 결과 진입
  3. 종합 결과 노출: 종합 점수(예 30/200), 등급(1~6급), 5각 레이더(발음·유창성·어휘·문법·내용 구성), AI 종합 피드백
  4. 문항 선택 시 문항별 점수·AI 피드백 모달: 답안 vs STT 대조, 추천 연습

예외·대체 흐름

  • 결과 미공개 상태 접근: "採点結果は公開後にご確認いただけます" 안내
  • 채점 지연·실패: 운영자 확인 대상 (재채점·재응시 티켓은 운영 판단)

관련 화면: 결과(종합), 문항별 AI 피드백 모달 (사용자 web)

데이터·입력: 응시 인스턴스ID, 영역별 점수, 등급, STT 텍스트, AI 피드백

정책·규칙: 정책5(결과 공개 후 노출), 정책6(재응시 한정). ※ 04-policy 정책5 참조

비고: 등급 1~6급 체계는 한국교육재단 협의 후 확정(TBD). 5각 레이더 차트는 결과 화면 핵심 시각 자산으로 발주 1단계.

4.9 FS-MY-04 / 인증서 다운로드

개요 결과 공개 후 마이페이지에서 인증서 PDF를 내려받는 기능. 인증서에는 kopass.* 도메인의 위변조 검증 URL이 포함되어, 제3자가 해당 URL로 진위를 확인할 수 있음. 인증서 발급은 브랜드 자산이므로 portal에 위치함(정책10).

선행조건: 결과 공개 완료된 응시 건 보유

기본 흐름

  1. 마이페이지 "인증서" 진입
  2. 발급 가능한 인증서 목록 노출
  3. "PDFダウンロード" 클릭 → PDF 생성·다운로드
  4. PDF 내 검증 URL 접속 시 검증 페이지에서 진위·발급정보 표시

예외·대체 흐름

  • 결과 미공개: 인증서 미노출
  • 검증 URL 접근(제3자): 로그인 없이 최소 정보만 노출, 위변조 시 무효 표시

관련 화면: 마이페이지 인증서 목록, 인증서 PDF, 위변조 검증 페이지

데이터·입력: 인증서ID, 검증 토큰/URL, 발급일, 시험명, 점수·등급

정책·규칙: 정책5(결과 공개 후), 정책10(브랜드 자산 소유권)

비고: 인증서 템플릿은 관리자 web에서 관리(FS-ADM-05). 검증 URL 노출 항목 범위는 개인정보 관점 검토 필요.

4.10 FS-MY-06 / 환불 신청·상태

개요 1차 오픈 환불은 모두 운영 수동 검토 처리(정책7). 응시자는 접수 기간 내 미응시 건에 한해 신청 가능하며, 응시 후 변심·결과 불만족은 신청 대상이 아님. 승인 시 PG 환불 호출 후 영업일 3~5일 내 환원.

선행조건

  • 로그인
  • 응시 윈도우 내 미응시 건 (응시 완료 건은 신청 불가)

기본 흐름

  1. 마이페이지 "환불 신청" 진입 → 대상 결제 건 선택
  2. 사유 입력 후 "返金を申請する"
  3. 접수 상태로 전환, 운영자 검토 대기 (FS-ADM-03)
  4. 승인 시 PG 환불 호출 → "환불 처리 중" → 완료
  5. 환불 처리 이메일 발송 (FS-NOTI-01)

예외·대체 흐름

  • 응시 완료 건: 신청 버튼 비활성, 불가 안내
  • 시스템 장애로 재응시 불가한 경우: 응시자 신청 없이 운영자가 직접 환불 처리
  • 반려: 사유와 함께 상태 갱신

관련 화면: 마이페이지 환불 신청·상태

데이터·입력: 결제 건ID, 환불 사유, 상태(접수/검토/승인/처리중/완료/반려)

정책·규칙: 정책7. 접수기간 내 미응시만 신청 가능, 운영 수동 검토, 영업일 3~5일 환원, 응시 후 변심·불만족 불가. ※ 04-policy 정책7 참조

비고: 자동 환불 분기는 라운드2. 환불 규정 안내 문안은 특정상거래법 표기와 연계.

4.11 FS-ADM-04 / 결과 공개 (관리자 연계)

개요 운영자가 관리자 web에서 특정 시험·회차의 채점 결과를 응시자에게 공개하는 기능. 이 클릭이 있기 전까지 응시자에게 결과·인증서가 노출되지 않음(정책5). 1차 오픈은 평가자 2차 채점 UI 없이 운영자가 수동 검수 후 공개함.

기본 흐름

  1. 관리자 web에서 대상 시험·응시자 목록 진입
  2. AI 1차 채점 결과 확인·수동 검수
  3. "결과 공개" 클릭
  4. 응시자 마이페이지 결과 노출 활성화 + 결과 공개 이메일 발송

예외·대체 흐름

  • 답안 초기화: 관리자만 가능. 초기화 시 해당 응시자 재응시 가능(정책5·정책6 연계)

관련 화면: 관리자 web 응시자 관리, 응시자 상세

정책·규칙: 정책5. 결과 공개 자동화(채점 후 N분)는 라운드2.

비고: 상세는 06-admin-feature-spec.md 참조.

4.12 부수 기능 (요지)

  • FS-MEM-04 비밀번호 재설정 / FS-MEM-05 계정 찾기: 이메일 링크 기반. 재설정 링크 유효시간 TBD.
  • FS-MEM-07 회원정보 수정 / FS-MEM-08 회원 탈퇴: 탈퇴는 경고 모달 경유. 탈퇴 시 데이터 삭제 정책은 라운드3(12-security.md).
  • FS-EXM-03 무료 샘플 3문항: 시험 상세에서 진입, 회원 게이트 발동. 샘플 결과는 본 시험 결제 유도 CTA와 연결.
  • FS-PAY-04 영수증: 마이페이지 결제 내역에서 PDF 다운로드. 특정상거래법·적격청구서 대응. PG 확정 후 항목 보완.
  • FS-MY-01 대시보드: 진행 중 시험, 미응시 D-day, 결과 발표 임박을 카드로 요약.
  • FS-CS-01~03 고객센터: 공지, FAQ 5카테고리, 1:1 문의 작성. 문의 답변은 관리자 web.
  • FS-NOTI-01 이메일 알림 6종: 결제완료·D-1·D-day·만료·결과공개·환불처리. 발송 트리거는 각 기능 흐름에 연결. 푸시는 라운드2.
  • FS-ADM-01/02/05 관리자 신규 메뉴: 회원(가입·결제·환불 집계), 결제 관리(결제 5상태 + 환불 처리), 인증서 템플릿 관리. 상세는 06-admin-feature-spec.md.
  • FS-ADM-06 문제 만들기 5영역 탭: 기존 문제 만들기에 말하기 5영역(짧은 문장 읽기·질문 대답·생활 속 대화·그림 설명·화용 과제) 탭 신설. 영역별 지시문·시간·배점 자동 적용, 응시자 폰 mock 라이브 프리뷰.

6. 비기능 요구사항

6.1 브라우저 지원

  • 데스크탑: Chrome·Safari·Edge 최신 버전
  • 모바일: 모바일 브라우저 (iOS Safari·Android Chrome)
  • 마이크 권한 필수, 헤드셋 권장 (응시 품질 확보)
  • IE·구형 브라우저 미지원

6.2 반응형

  • 브레이크포인트: 데스크탑 ≥1024px / 태블릿 768~1023px / 모바일 ≤767px
  • 시험 상세·결제: 데스크탑 우측 사이드바, 모바일 하단 sticky CTA
  • 마이페이지: 데스크탑 좌측 메뉴, 모바일 하단 탭
  • 응시 화면: 데스크탑 2단(좌 문제/우 응시), 모바일 1단
  • 관리자 web은 데스크탑 전용 (1차 오픈 모바일 미지원)

6.3 다국어(i18n)

  • UI 1차 언어: 일본어
  • 시험 콘텐츠: 한국어 (지시문 일부 일본어 병기)
  • 일본어 라벨 네이티브 검수 필요 (TBD)
  • 폰트: 한·일 동시 지원(Pretendard 계열, :lang(ko) 분기 고려)

6.4 접근성

  • 키보드 포커스 이동·포커스 표시
  • 폼 요소 라벨 연결
  • 1차 오픈은 기본 수준. 세부 WCAG 준수 수준은 라운드 이후 상향 검토

6.5 성능·보안

  • 라운드3에서 별도 상세 명세 (12-security.md)
  • 회원 게이트는 UI 장치이며 실제 접근 통제는 API 인증 토큰 검사로 보장
  • 개인정보 최소 수집, 일본 개인정보보호법 대응은 라운드3
