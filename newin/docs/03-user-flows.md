# 유저 플로우 + 서비스 웹사이트 구조 (1차 오픈, Light)

> 본 문서는 1차 오픈 범위에 한정한 가벼운 골격이다. 환불 분기·재응시 티켓·딥링크 같은 무거운 동선은 *1차 오픈 범위 외* 섹션에서 목록만 둔다.

## 한 줄

**1차 오픈은 *가입 → 접수 → 결제(★ portal) → 응시(◇ 사용자 web, 새 창) → 결과(◇ 사용자 web) → 인증서·PDF(★ portal)* 한 동선으로 충분하다.**

## 이 한 줄을 지지하는 이유 5개

1. **MaroPass portal(★ 신규) + 터치클래스 자산 재사용(◇)** 구조이므로 동선도 portal·사용자 web 둘 사이의 새 창 점프 한 번만 둔다 (정책 10).
2. **관리자 측은 기존 터치클래스 관리자 web을 그대로 사용**하므로 동선은 MaroPass 메뉴 5개(회원·결제·환불 승인·결과 공개·인증서 템플릿)에만 추가 동선이 붙음. 시험 관리·문제 은행·응시자 관리는 기존 그대로.
3. **새 창은 UI 수단**이고 본질은 소유권·책임 분리. 학습자가 응시 중에도 portal에서 결제 내역·PDF를 병행 확인 가능.
4. **결과 확인은 사용자 web(학습 자산), PDF 다운로드(영수증·인증서·결과지)는 portal(브랜드 자산)** 로 명확히 분리 — 위변조 검증 URL이 `kopass.*` 도메인이어야 신뢰 성립.
5. **1차 오픈은 데스크탑·모바일 웹만**. 앱·푸시 알림은 라운드 2 이후이므로 플로우도 웹 기준만 잡는다. 라운드 2 이후 터치클래스 app 안에 응시 클라이언트 wrapping 가능.

## 2. ★ MaroPass portal — 사이트맵 (신규)

```
MaroPass portal (결제·환불·인증서)
│
├─ 회원 관리
│   ├─ 회원가입 (이메일 + LINE·Google·Apple 소셜)
│   ├─ 로그인 (SSO 토큰으로 사용자 web 자동 로그인)
│   └─ 계정 찾기 / 비밀번호 재설정
│
├─ 상품·결제
│   ├─ 시험 목록 (검색·필터)
│   ├─ 시험 상세 (바로 구매하기·무료 샘플 응시 CTA — 회원 게이트 적용)
│   └─ 결제 (일본 PG 1개사 — 신용카드 + QR)
│
├─ 마이페이지 (★ portal 핵심)
│   ├─ 대시보드 (응시 가능 시험·최근 결과·인증서)
│   ├─ 응시할 시험 → "응시하기" 버튼 ──[window.open, 새 창]──→ ◇ 사용자 web home
│   ├─ 결과 보기 진입 → 사용자 web 결과 화면으로 이동
│   ├─ ★ 인증서 PDF 다운로드 (kopass.* 위변조 검증 URL)
│   ├─ ★ 결과지 PDF 다운로드
│   ├─ ★ 결제 내역 (영수증 PDF — MaroPass 사업자 정보로 발행)
│   ├─ ★ 환불 요청
│   ├─ 1:1 문의 (작성·답변 확인)
│   └─ 회원정보 수정 / 탈퇴
│
└─ 고객센터
    ├─ 공지 / FAQ
    └─ 1:1 문의
```

> URL 정의·메뉴 라벨 일본어 표기·페이지 분리 단위는 `05-screen-inventory`·`07-ia.md` 참조.

## 3. ◇ 터치클래스 사용자 web — MaroPass 응시 영역 추가 (기존 web 재사용)

```
터치클래스 사용자 web (MaroPass 응시 영역, 새 창 진입)
│
├─ ★ home — MaroPass 결제분 시험 목록 (응시 대기 + 응시 완료) — 신규 라우팅
│
├─ 응시 진입 (응시 대기 카드의 "응시 시작" 클릭) — 기존 인프라 활용
│   ├─ ◇ 본인 확인 (이름·이메일·시험명 표시 + 체크박스) — 기존
│   ├─ ◇ 환경 점검 게이트 (마이크·볼륨·5초 리플레이 3단계) — 기존
│   └─ ★ MaroPass 응시 클라이언트 (말하기 IBT 5영역, 데스크탑 2단 / 모바일 1단) — 신규
│
└─ ★ 결과 확인 (응시 완료 카드의 "결과 보기" 클릭, 운영자 공개 후 활성화) — MaroPass 채점 포맷 신규
    ├─ 점수표 (5축) + 5각 레이더 차트
    ├─ 1급~6급 진단 등급 + 종합 코멘트
    ├─ 문항별 AI 피드백 (STT 비교·추천 연습)
    └─ "인증서·결과지 PDF 다운로드는 MaroPass portal 마이페이지에서" 안내 링크
```

> 사용자 web은 **새 창**으로 열리므로 portal이 백그라운드에 유지됨 — 본질은 학습 자산(사용자 web) vs 브랜드 자산(portal) 소유권 분리.
> 환경 점검 통과 이력은 localStorage 30일 유효 (정책 1 보조).
> 라운드 2 이후 터치클래스 app 네이티브 쉘 안에 응시 클라이언트 wrapping 가능 (코드 베이스 동일).

## 4. ◇ 터치클래스 관리자 web — 대메뉴 (기존 + MaroPass 메뉴 5개 추가)

**기존 터치클래스 관리자 web을 그대로 사용**. 운영자는 익숙한 시험 관리·문제 은행·응시자 관리 워크플로우를 유지. MaroPass portal 데이터를 받는 신규 메뉴 5개(★)만 추가.

1. **★ MaroPass 회원 (신규)** — MaroPass portal 가입·결제·환불 데이터 집계. 단일 화면에서 회원 정보·결제 이력·응시 이력 확인
2. **★ MaroPass 결제 관리 (신규)** — 결제 내역, 영수증 발행
3. **★ MaroPass 환불 승인 (신규)** — 환불 요청 검토·승인 (정책 7)
4. **★ MaroPass 결과 공개 (신규)** — AI 1차 채점 후 응시자 노출 트리거 (정책 5)
5. **★ MaroPass 인증서 템플릿 (신규)** — 위변조 검증 URL `kopass.*` 도메인 템플릿
6. **◇ 시험 관리 (기존)** — 시험 등록·공개, 응시 조건, 시험지 구성
7. **◇ 문제 은행 · 문제 만들기 (기존 + ★ MaroPass 말하기 5영역 탭 신설)** — 객관식·주관식·말하기 1형·2형은 기존, MaroPass 5영역 탭이 신규
8. **◇ 평가 및 채점 (기존)** — 응시자별 채점, AI STT 결과 확인·수정, 점수 보정
9. **◇ 1:1 문의 (기존)** — 응시자 문의 답변
10. **◇ 통계 및 분석 (기존)** — 응시 통계, AI 평가 품질 (1차는 KPI 4종만)
11. **◇ 시스템 설정·보안 (기존)** — 약관, 활동 로그 (2FA는 라운드 3)

> 향후 핵심은 **LLM 자동 생성 보조** (문제 만들기 깊이 — 라운드 2).

## 5. 1차 오픈 핵심 플로우 (응시자 5단계)

### 5.1 STEP 1 — 진입 (★ MaroPass portal)
```
방문자 → 시험 목록 → 시험 상세
  ↓ 바로 구매하기 / 무료 샘플 응시 클릭
회원 게이트 (정책 9):
  비로그인 → 가입 페이지 (intent=sample|purchase&return=...)
  로그인 → 다음 단계
```

### 5.2 STEP 2 — 결제 (★ MaroPass portal)
```
주문서 작성 → 일본 PG (신용카드 / QR) → 결제 완료
  ↓ 자동 메일: 결제 확인 + 영수증 PDF 안내 (MaroPass 사업자 정보)
portal 마이페이지 응시할 시험에 D-day 카드 표시
```

### 5.3 STEP 3 — 대기 (★ MaroPass portal, 결제 후 14일 윈도우)
```
portal 마이페이지에서 D-day 확인
  ↓ D-1, D-day 자동 리마인더 메일
14일 경과 시 자동 만료 (환불 없음, 재응시 없음)
```

### 5.4 STEP 4 — 응시 (◇ 터치클래스 사용자 web, 새 창)
```
portal 마이페이지 "응시하기" 클릭
  ↓ window.open() → 사용자 web home 새 창 (팝업 차단 시 허용 안내)
  ↓ SSO 토큰 자동 로그인 (portal 로그인 세션 공유)
사용자 web home → MaroPass 시험 카드 "응시 시작"
  ↓ ◇ 본인 확인 (이름·이메일·시험명 + 체크박스) — 기존 인프라
  ↓ ◇ 환경 점검 게이트 (마이크·볼륨·5초 리플레이, 30일 캐시 적중 시 스킵) — 기존
  ↓ ★ MaroPass 응시 클라이언트 (말하기 IBT 5영역, 자동 녹음·타이머) — 신규
시험 종료 → AI 1차 자동 채점
```

### 5.5 STEP 5 — 결과 (◇ 사용자 web) · 인증서·PDF (★ portal)
```
운영자가 관리자 web에서 ★ MaroPass 결과 공개 처리
  ↓ 자동 메일: 결과 공개 안내
응시자 → ◇ 사용자 web: 결과 확인 (점수표·5각 레이더·등급·AI 피드백)
       → ★ portal 마이페이지: 인증서 PDF (kopass.* 위변조 URL) · 결과지 PDF · 영수증 PDF 다운로드
```
