# Phase 로드맵

## 한 줄

**1차 오픈(Phase 1)은 *말하기 진단 + ★ MaroPass portal 신규 + ◇ 터치클래스 자산 재사용(사용자 web·관리자 web)* 한 동선으로 끝낸다. 그 외는 라운드 2 이후.**

## 이 한 줄을 지지하는 이유 5개

1. 데드라인이 **2026년 9월 서비스 개시**로 박혀 있어 범위 확대는 일정 위험으로 직결된다.
2. **기존 터치클래스 자산 재사용**(사용자 web 응시 인프라·관리자 web 시험 관리)으로 신규 개발 표면적·일정 위험을 1/3로 축소. 신규 개발은 MaroPass portal + 특화 영역에만 집중.
3. 관리자 측은 기존 터치클래스 관리자 web을 그대로 사용하므로 1차 오픈 신규는 *MaroPass 메뉴 5개*(회원·결제·환불 승인·결과 공개·인증서 템플릿) + *말하기 5영역 탭*뿐. 시험 관리·문제 은행·응시자 관리는 기존 그대로.
4. 평가자(2차 채점)·딥링크·푸시 알림·장바구니 같은 무거운 기능은 1차 오픈 후 운영 데이터를 보고 결정하는 게 합리적이다.
5. 라운드를 작게 끊으면 갭 결정이 빠르게 본문에 반영되어 문서가 늘 최신 상태로 유지된다.

## Phase 1 — 1차 오픈 (2026.9 목표)

### 1. ★ MaroPass portal (서비스 사이트) — 회원·결제·환불·인증서 (신규)

| 영역 | 1차 오픈 포함 |
|---|---|
| 회원관리 | 가입(이메일·LINE·Google·Apple) / 로그인 / 비밀번호 재설정 / 계정 찾기 |
| 상품·결제 | 시험 목록·상세, 무료 샘플 응시 진입, 일본 PG 1개 연동 결제 (신용카드 + QR) |
| 회원 게이트 | 무료 샘플·바로 구매 클릭 시 로그인 필수 (정책 9). 비로그인 → 가입 페이지 |
| 마이페이지 | 구매 시험·응시할 시험·결제 내역·환불 신청·**영수증/인증서/결과지 PDF 다운로드**(kopass.* 위변조 URL)·정보 수정 |
| 응시 진입 | **응시하기 → window.open()으로 새 창** (터치클래스 사용자 web home, 정책 10) + SSO 토큰 자동 로그인 |
| 고객센터 | 공지·FAQ·1:1 문의 |
| 알림 | 이메일 5종 (결제 완료·D-1·D-day·만료·결과 공개·환불 처리). 푸시는 라운드 2 |
| 디바이스 | 데스크탑·모바일 웹만. iOS/Android 앱은 라운드 2 (portal은 웹 단독 유지) |

### 2. ◇ 터치클래스 사용자 web — MaroPass 응시 영역 추가 (기존 web 재사용)

| 영역 | 1차 오픈 포함 |
|---|---|
| ★ home (신규) | MaroPass 결제분 시험 목록 (응시 대기 + 응시 완료). 기존 사용자 web의 다른 콘텐츠와 분리 라우팅 |
| ◇ 응시 진입 (기존) | 본인 확인(이름·이메일·시험명) → 환경 점검 게이트(마이크·볼륨·5초 리플레이, 30일 캐시) |
| ★ MaroPass 응시 클라이언트 (신규) | 말하기 IBT 5영역, 데스크탑 2단 / 모바일 1단. 자동 녹음, 타이머, 자동 저장·전송. 라운드 2 이후 터치클래스 app 안에 wrapping 가능 |
| ★ 응시 결과 확인 (신규) | MaroPass 채점 포맷 — 점수표·5각 레이더·1급~6급 등급·문항별 AI 피드백 (운영자 결과 공개 후) |
| ★ PDF 다운로드 안내 (신규) | "인증서·결과지 PDF는 MaroPass portal 마이페이지에서" |

### 3. ◇ 터치클래스 관리자 web — MaroPass 메뉴 5개 추가 (기존 web 재사용)

**기존 터치클래스 관리자 web을 그대로 사용**. 시험 관리·문제 은행·응시자 관리·평가/채점·통계·시스템 설정은 기존(◇). MaroPass portal 데이터를 받는 신규 메뉴 5개(★)만 추가.

| 영역 | 1차 오픈 포함 |
|---|---|
| ★ MaroPass 회원 (신규) | MaroPass portal 가입·결제·환불 데이터 집계. 회원 정보·결제·응시 이력 |
| ★ MaroPass 결제 관리 (신규) | MaroPass 결제 내역·영수증 발행 |
| ★ MaroPass 환불 승인 (신규) | MaroPass 환불 요청 검토·승인 (정책 7) |
| ★ MaroPass 결과 공개 (신규) | AI 1차 채점 후 응시자 노출 트리거 (정책 5) |
| ★ MaroPass 인증서 템플릿 (신규) | kopass.* 위변조 URL 도메인 템플릿 |
| ◇ 시험 관리 (기존) | 시험 등록·공개·응시 조건 설정 |
| ◇ 문제 은행 (기존 + ★ MaroPass 말하기 5영역 탭) | 객관식·주관식·말하기 1형·2형은 기존, MaroPass 5영역 탭이 신규 |
| ◇ 평가/채점 (기존) | AI 1차 자동 채점. 운영자 보정·STT 확인 |
| ◇ 1:1 문의 (기존) | 답변 처리 |
| ◇ 통계 (기존) | KPI 4종만 1차 적용 |
| ◇ 시스템 설정 (기존) | 약관·활동 로그. 2FA는 라운드 3 |

> 향후 핵심: **문제 만들기 LLM 자동 생성 보조** (라운드 2). 평가자 UI(3단 분할·STT 싱크·2인 교차 채점)는 Premium 도입 시점에 종속.

### UI 공통 적용

- 한국어/日本語 i18n 토글 (localStorage 동기화 + `data-i18n` / `data-i18n-html` 패턴)
- 햄버거 메뉴 (≤768px) + 풀너비 드로어 + 좌우 20px 패딩
- 반응형 베이스 리셋 (`*{box-sizing}` + `html,body{overflow-x:hidden;max-width:100vw}` + `img,svg{max-width:100%}`)
- 페이지 간 링크 전체 연결 (GNB·CTA·sidebar·breadcrumb)
- 일관 톤 (마케팅·홈은 그라디언트 / 응시·결제는 차분)
- 응시 화면 1080px shell 가운데 정렬 + 상하 70/30 분할
- 메인 홈 FAQ 5개 아코디언 + 한·일 답변 더미 + `<strong>` 강조
- 로그인 진입은 `ui-auth.html#login` 해시로 자동 탭 활성화
